C++ decltype

decltype

decltype 是 C++11 新增的一个用来推导表达式类型的关键字. 和 auto 的功能一样, 用来在编译时期进行自动类型推导. 引入 decltype 是因为 auto 并不适用于所有的自动类型推导场景. 也可以将 decltype 看作是 sizeof 运算符的另一种形式, 因为两者都不会真正计算值, 只充当一种编译期工具.

decltype 对比 auto:

  • auto 根据 = 右边的初始值推导出变量的类型, decltype 根据 exp 表达式推导出变量的类型.

  2. 对于其他的情况, 包括 E 是一个加括号的 id-expression (如 (x), (s.field) 这样的表达式), C++ 会完全隐去 E 类型中的引用, 使得引用完全不可见. 此时 decltype(E) 先提取 E 的不含引用的类型 T，然后根据 E 的三种值类别来确定 decltype(E) 的结果: 当 E 的值类别是 prvalue, decltype(E) 返回 T; 当 E 的值类别是 lvalue, decltype(E) 返回 T&; 当 E 的值类别是 xvalue, decltype(E) 返回 T&&. 例如:

decltype 和值类别

对于推导规则中的第一种运算, 我们称之为 variable decltype, 因为这种 decltype 返回的是变量 (或者字段) 定义时的类型.

对于推导规则中的第二种运算, 我们称之为 expression decltype. 我们可以通过该运算, 推导出表达式的值类别. 如果 decltype((E)) 得到的是非引用类型 T, 那么 E 的值类别是 prvalue; 如果是 T&, 那么 E 的值类别是 lvalue; 如果是 T&&, 那么 E 的值类别是 xvalue.

此时可以将值类别和 expression decltype 的引用类型绑定起来:

expression decltype 值类别
非引用类型 T prvalue
lvalue 引用 T& lvalue
rvalue 引用 T&& xvalue

截至 C++17, 一个类型为 T 的 prvalue 是一个可以生成一个类型为 T 的对象的抽象配方 (recipe). 值得注意的是, 一个 prvalue 并不对应一个你程序中真实存在的对象, 也不意味着构造函数已经调用过. 字面常量如 5, true, nullptr 和 enum 标签都是 prvalue, 因为可以用他们来初始化对象和操作数.

auto s = std::string("hello world"); 中的 std::string("hello world") 是一个 prvalue. 但是, 得到这么一个 prvalue 的时候, 并没有真的创建一个字符串, 甚至 std::string(const char*) 这个构造函数也没有调用. 这一点是合理的, 因为设想一下如果这个 prvalue 是一个构造好的对象, 那么 s 就必须通过移动构造来初始化. 所以在这个例子中, s 是直接由参数 "hello world" 构造初始化的. 在语言标准里, 一个最终由 prvalue 初始化的对象被称为这个 prvalue 的 result object, 而被用于初始化 result object 的值被称为 prvalue 的 result.

编译器会尽可能地推迟这个将 prvalue 变成一个对象的过程 (materializing), 以避免不必要的移动和拷贝构造. 尽管如此, 即使 prvalue 的值最后没有被使用, 它仍然需要被 materialize, 所以推迟 materialize 并不能省略所有的构造, 它只能避免移动和拷贝构造.

一个 glvalue 代表了一个程序中真实存在的对象, 它可以唯一地确定一个对象, 它在内存中有确定的地址.

一个 xvalue 是一个即将被丢掉的值. 比如, 一个在表达式中被创建的临时变量的生命周期会在表达式的末尾结束, 此时它将被销毁.

一个 lvalue 就是一个非 xvalue 的 glvalue.

一个 rvalue 就是一个 prvalue 或者 xvalue.

由于 prvalue 在初始化其 result object 之前必须先被 materialize, 所以 prvalue 必须是一个完整类型, 而不能是前向声明的一个 struct. 类似地, prvalue 也不能是纯虚类, 因为纯虚类并不能实例化. 另外, 有意义的被 const 和 volatile 修饰的 prvalue 只能是类和数组. 比如, 一个 const int 类型的 prvalue 在初始化一个 int 变量的时候, 和普通的 int 类型的 prvalue 没有任何区别.

内建运算符对其操作数有一定的要求, 并且返回特定的值类别. 比如, 内建二元运算符 + 要求两个操作数都是 prvalue, 并且返回 prvalue. 当然, 即使 + 要求操作数都是 prvalue, 你仍然能让其中一个操作数是 lvalue, 比如 x + 1. 这是因为, 一个非函数或数组的 glvalue 都可以通过一种称为 lvalue-to-rvalue conversion 的转换变成一个 prvalue. 根据上文所述, prvalue 实际上只是一个用来初始化对象的配方. 当从一个非类的 glvalue, 比如 int 转换到 prvalue 的时候, 你得到了这么一个配方: 根据老的 int 的值初始化一个新的 int; 当从一个类的 glvalue 转换到 prvalue 的时候, 这个配方就是: 根据现有的 glvalue, 拷贝或者移动构造一个新的类实例.

一个表达式的值类别限定了它能绑定到哪些引用上, 如果 T 是一个非引用类型, 那么:

  • T& 只能绑定 lvalue
  • T&& 只能绑定 rvalue
  • const T& 可以绑定任意的值类别, 但是如果既存在接受 const T& 的函数又存在接受 T&& 的函数时, 那么重载决议会优先使用 T&& 的函数而不是 const T&.

当一个引用绑定到 prvalue 的时候, 这个 prvalue 会先被 materialize 成为一个临时对象. 但是, 一般来说, 一个临时对象在表达式结束的地方就会被销毁, 这就会导致这个引用成为悬垂引用. 所以为了避免这种情况, C++ 会延长绑定到引用上的临时对象的生命周期, 使得直到引用的作用域结束之前临时对象都是有效的.

下面介绍 expression decltype 的小技巧, 之后我们将 expression decltype 简称为 exprtype.

我们将表达式分成三种情况: 特例、有名值和无名值. 特例包含字符串字面量和函数, 它们的 exprtype 永远是 T&. 有名值包括变量 (x)、对象的成员变量 (s.f) 和数组元素 (a[i]), 它们的 exprtype 一般都是 T&, 除了几种情况: s 不是 lvalue 而且销毁 s 会导致 s.f 也被销毁, 此时 s.f 的 exprtype 是 T&&;  a 不是 lvalue, 此时 a[i] 的 exprtype 是 T&&. 最后, 无名值的 exprtype 和它们的 C++ 类型相同. 无名值分为两种: 显式地具有类型的值, 即类型转换表达式、函数调用和其他的和内建运算符有关的情况.

下面是所有情况的详细解释:

特例 lvalue: 字符串字面量、函数和函数的引用的 exprtype 永远都是 lvalue 引用.

E decltype((E))
“hello” const char(&)[6]
getpid int(&)()
static_cast<int(&)()>(getpid) int(&)()
std::move(getpid) int(&)()

由于字符串字面量是内存中的一个 char 数组, 而函数是内存中的指令, 它们都不是 prvalue, 所以让它们具有非引用的 decltype 是没有意义的. 另外, 它们的生命周期贯穿整个程序, 所以允许 rvalue 引用来移动它们也是没有意义的. 所以, 只有 lvalue 引用是有意义的.

有名值: 如果 E 是变量 (x)、类或联合体的成员变量 (s.f) 或者是数组的元素 (a[i]), 那么它唯一地对应了一个在内存中存在的、已经构造好的对象, 从而不能是一个 prvalue. 因此, E 的 exprtype 必须是一个引用. 在这种情况下如果 E 是一个 exprtype 为非 lvalue 引用 (即该对象的值类别是 lvalue) 的对象的成员或者是数组的元素, 而且 E 的 variable decltype 不是引用, 那么 E 的 exprtype 是 rvalue 引用; 否则 E 的 exprtype 是 lvalue 引用.

v int&
vref int&

这里需要重点解释一个比较难处理的情况: 像 S::data_member 这么一个非静态的成员变量为什么永远是 lvalue? 这是因为在 S 的一个成员函数内, S::data_member 等价于 (*this).S::data_member, 它显然是 lvalue 引用. 在 S 的外部, 对 S::data_member 求值是非法的, 但是 decltype((S::data_member)) 并不会对它求值. 甚至这样的语句都是合法的: decltype(S::data_member + 5).

显含类型的值: 当一个无名值显含类型, 那么其 exprtype 就是这个表达式的类型. 更具体地来说:

  • 所有的字面量 (除了字符串字面量)、enum 标签、this 指针和其他非类型地模板参数的 exprtype 都和它们本身的类型完全相同:

    E decltype((E))
    true bool
    5 int
    ‘A’ char
    nullptr std::nullptr_t
    enum 标签 对应的 enum 类型
    成员函数中的 this 指针 T* 或者 const T*

  • 函数调用的返回值、包括重载的运算符, 其 exprtype 都和函数原型中的返回值完全相同:

    E decltype((E))
    std::terminate() void
    std::to_string(5) std::string
    co_await a decltype(a.await_resume())
    std::string(“hello”) + " world" std::string

  • 强制类型转换和类型转换的 exprtype 都是它们的目标类型:

    E decltype((E))
    double(1) double
    static_cast<int&>(v) int&
    static_cast<int&&>(v) int&&
    std::string{“hello”} std::string

  • new 表达式的 exprtype 就是新创建对象的指针类型:

    E decltype((E))
    new T T*

内建运算符: 内建运算符的 exprtype 和你在自定义运算符重载的时候的类型非常相似.

三目运算符 E1 ? E2 : E3 是一个例外, 因为它不提供重载. 三目运算符会尝试统一 E2 和 E3 的 exprtype, 返回一个共同的引用类型. 例如: 如果 E2 和 E3 的 exprtype 都是 int&, 则返回一个 int&; 如果 E2 和 E3 分别是 int 和 int&, 则没法统一, 最后返回一个 int.

指针解引用运算符 (*p, 其中 p 的类型是 T*) 的 exprtype 是 T&.

sizeof(E) 的 exprtype 是 std::size_t.

取地址运算符 (&a, 其中 a 的非引用类型是 T), 其 exprtype 是 T*.

其他内建运算符的 exprtype 易判断, 就不再赘述.
