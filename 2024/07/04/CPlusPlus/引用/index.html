rvalue 引用

类型为 T 的 rvalue 引用的语法可以表示为 T&&, rvalue 引用引用可移动值, 即其内容在使用后无需保留的值 (例如临时值). 由于关键是移动绑定到 rvalue 引用的值, 因此不会将 const 和 volatile 限定符 (也称 cv 限定符) 应用到 rvalue 引用.

rvalue 引用绑定到 rvalue. 实际上, 在进行重载决策时, rvalue 首选绑定到 rvalue 引用而不是 lvalue 常量引用 (T const &). 但是, rvalue 引用不能绑定到 lvalue, 因为如前所述, rvalue 引用引用的是假定其内容不需要保留的值.

移动语义

rvalue 引用支持 “移动语义” 的实现, 这可以显著提高应用程序的性能. 利用移动语义, 你可以编写将资源 (如动态分配的内存) 从一个对象转移到另一个对象的代码.

若要实现移动语义, 通常可以向类提供 “移动构造函数”, 或者提供移动赋值运算符 (operator=).

已命名 rvalue 引用 / 未命名 rvalue 引用

引用折叠

函数参数推导时先会推导出其模板参数类型, 然后使用引用折叠规则.

下表汇总了模板自变量类型推导的引用折叠规则:

展开类型 折叠类型
T& & T&
T& && T&
T&& & T&
T&& && T&&

如果传递给函数的表达式的值类别是 rvalue, 则编译器将函数参数类型推导为 rvalue 引用.

  • 例如, 假设将类型为未命名 rvalue 引用 (假设该表达式的类型为 X&&) 的表达式传递给采用类型 T&& 作为其参数的函数模板: 此时模板参数推导出 T 为 X, 则该函数参数的类型为 X&&.

如果传递给函数的表达式的值类别是 lvalue, 则编译器将函数参数类型推导为 lvalue 引用.

  • 例如, 假设将类型为 X 或 const X 的表达式传递给采用类型 T&& 作为其参数的函数模板: 此时模板参数推导出 T 为 X& 或 const X&, 则该函数参数的类型为 X& && 或 const X& &&, 然后使用引用折叠得到该参数的类型为 X& 或 const X&.
