C++ auto

auto

在 C++ 中, 变量声明通常需要显式指定其类型, 例如 int、double、char, 在 C++ 中使用 auto 关键字时, 它能够让编译器根据变量初始化的表达式自动推导出变量的类型. auto 关键字的使用在 C++11 中引入, 之后的标准中也得到了进一步增强和扩展.

通过上面的例子可以看出, auto 实际上只是一个占位符, auto 关键字的类型推断是在编译期进行的, 在编译器期间, auto会被真正的类型所替代, 因此使用 auto 不会对程序的运行时效率造成影响. 并且由于编译器通过初始化表达式推导出变量的类型, 所以 auto 关键字必须初始化.

使用 auto 的原因如下:

  1. 上下一致性.
    在某些地方, 如果你希望编译器去自己根据上下文环境确定一个变量的类型而用了auto, 那么你在每个类似的地方都要使用它, 使用两种不同的编码习惯是不合适的.
  2. 可维护性.
    使用 auto 可以改善代码的可维护性. 例如: 当 int 不足以存储数据或其他原因导致要改为 long 时, 只需要把上面的 std::vector<int> 改为 std::vector<long> 即可, 其他变量的类型变化就交给编译器和 auto 来处理, 这使得代码便于维护.

限制

  1. auto 只能在局部作用域中使用, 不能用于全局变量或类的成员变量的声明.
  2. 不支持数组类型推导, 不能使用 auto 推导数组类型, 因为数组类型不会自动转换为指针类型.
  3. 不支持函数参数类型的推导, 不能在函数参数的声明中使用 auto 进行类型推导.
  4. 不能用于非静态成员变量, 因为非静态成员变量必须在类声明时就确定类型:

使用

  1. 初始化 auto 声明的变量时应该使用不戴花括号的拷贝赋值操作符 =, 如果使用以下方式可能会出现问题:

    • 假如你有一个类 SomeClass, 想要使用 auto x(SomeClass()) 来得到一个 SomeClass 类型的 x, 此时 auto 推导的结果其实是一个函数声明 auto x(SomeClass(*)()).
    • 假如你用 auto x{something} 来初始化, 可能会得到一个错误的类型推演, 此时 auto 推导的结果实际上是 initializer_list<SomeType>, 同样, 如果写成 auto x = {something}, auto 推导的结果也是 initializer_list<SomeType>.

    所以只有使用 = 才是最稳妥的策略, 即 auto x = something.
