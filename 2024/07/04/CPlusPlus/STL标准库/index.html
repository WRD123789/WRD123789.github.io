C++ STL 标准库

STL 标准库

STL 体系结构

C++ 标准库 (C++ Standard Library) 是一组预定义的函数、类和对象, 这些内容都包含在 C++ 标准中. 它提供了许多常用的功能, 如输入/输出、字符串处理、容器、算法、日期和时间等.

标准库被设计为可移植、高效且可靠的工具, 可以在不同的 C++ 编译器和平台上使用.

标准库的组成部分被组织在不同的头文件中, 通过 #include 指令来引入使用.

标准模板库 (Standard Template Library, STL) 是 C++ 标准库的一个重要组成部分, 它提供了一组模板类和函数, 用于实现通用的数据结构和算法.

STL 的设计目标是提供一组高效、通用和灵活的工具, 可以在不同情况下重用.

STL 包括以下主要组件:

  • 容器 (Containers): 容器是用于存储和管理数据的数据结构. STL 提供了多种类型的容器, 如向量 (vector)、链表 (list)、队列 (queue)、栈 (stack)、映射 (map) 等. 每种容器都有其特定的特点和适用场景. 从实现的角度来看容器是一种类模板.
  • 算法 (Algorithms): 算法是对数据执行操作的一组功能. STL 提供了许多通用的算法, 如排序、搜索、变换等. 这些算法可以用于不同类型的容器, 并且它们的实现是高度优化的, 以便在各种情况下获得最佳性能. 从实现的角度来看, STL 算法是一种函数模板.
  • 迭代器 (Iterators): 迭代器用于在容器中遍历元素. 它们提供了一种统一的方式来访问容器中的元素, 而不必了解容器内部的具体实现细节. 从实现角度来看, 迭代器是一种将 operator*、operator->、operator++、operator-- 等指针操作予以重载的类模板.
  • 仿函数 (Functors): 也称为函数对象 (Function Objects), 是可调用的对象, 可以像函数一样使用. 从实现角度来看, 仿函数是一种重载了 operator() 的类或者类模板.
  • 适配器 (Adapters): 适配器是一种用于改变容器或迭代器接口的机制.
  • 分配器 (Allocators): 分配器用于管理内存分配和释放, 是 STL 内部用于容器和算法的内存管理机制. 默认情况下, STL 使用默认分配器, 但你也可以为容器提供自定义的分配器以满足特定需求. 从实现上来看, 分配器是一个实现了动态配置空间、空间管理、空间释放的类模板.

STL 容器

容器是存储其他对象的对象. 被存储的对象必须是同一种类型的.

STL 的容器可以分为顺序容器、关联容器和无序关联容器.

顺序容器

顺序容器是一类用于存储和管理元素的数据结构, 其特点是元素在容器中的顺序与它们被插入的顺序相同, 其元素按严格的线性顺序排列, 即存在第一个元素、最后一个元素, 除第一个元素和最后一个元素外, 每个元素前后都分别有一个元素.

STL 顺序容器包括 deque、forward_list、list、queue、priority_queue、stack 和 vector.

vector

vector 是一种封装了动态数组的顺序容器.

vector 具有以下特点:

  • 动态数组大小: vector 允许你在运行时动态地添加或删除元素, 而不需要提前指定数组的大小.
  • 连续存储: vector中的元素在内存中是连续存储的, 这有助于实现高效的随机访问.
  • 随机访问: 你可以使用索引来访问任何元素, 这使得 vector 适用于需要快速随机访问的场景.
  • 尾部操作: 在 vector 的末尾添加或删除元素的操作通常是高效的, 因为不需要移动大量的元素.
  • 自动扩展: 当 vector 的容量不足以存储新的元素时, 它会自动扩展以容纳更多的元素, 通常以两倍的容量增加.
  • 迭代器支持: vector 支持正向迭代器和逆向迭代器, 使得你可以遍历容器中的元素.

deque

deque 也是一种顺序容器, 它允许在两端高效地进行元素插入和删除操作, 同时还支持随机访问.

deque 具有以下特点:

  • 双端操作: deque 支持在其头部和尾部高效地添加和删除元素. 这使得它适用于需要频繁插入和删除操作的情况, 而不仅限于 vector 末尾的操作.
  • 连续存储块: 与 vector 类似, deque 也使用了一系列连续的存储块来存储元素, 从而实现了较好的随机访问性能.
  • 动态增长: deque 会动态地增加内部存储块, 以适应新的元素. 这使得它适合于动态变化的数据大小.
  • 随机访问: 与 vector 类似, deque 也支持通过索引进行随机访问. 这使得你可以在 O(1) 的时间内访问元素.

list

list 是一个双向链表, 它能够支持高效的插入和删除操作, 但在随机访问方面的性能较差.

list 具有以下特点:

  • 双向链表: list 中的元素以双向链表的形式存储, 每个元素都包含指向前一个元素和后一个元素的指针. 这使得在中间位置插入和删除元素变得高效, 而不需要移动其他元素.
  • 无随机访问: list 不支持通过索引进行随机访问, 因为要访问中间元素, 必须从链表的开头或结尾开始遍历.
  • 不连续存储: 与 vector 和 deque 不同, list 中的元素并不是连续存储的. 这可能导致在一些情况下内存使用效率较低.

forward_list

forward_list 是一个单链表, 它的每个节点都只链接到下一个节点, 而没有链接到前一个节点. 因此 forward_list 只需要正向迭代器, 而不需要双向迭代器.

相比 list , 由于只需要存储一个指针, 所以 forward_list 所占用的内存更小.

queue

queue 是一个容器适配器, 用于实现队列数据结构. 它是基于其他STL 顺序容器 (通常是 deque) 实现的, 即它内部使用其他容器来存储数据, 对外提供队列操作的接口.

priority_queue

priority_queue 是一个容器适配器, 用于实现优先队列数据结构. 优先队列是一种特殊的队列, 其中元素具有优先级, 每次出队操作都会返回具有最高优先级的元素.

priority_queue 具有以下特点:

  • 优先队列操作: priority_queue 支持优先队列的基本操作, 包括插入元素和出队操作. 每次出队操作都会返回具有最高优先级的元素.
  • 堆实现: priority_queue内部使用堆 (通常是最大堆, 堆内部结构可以使用多种方式来实现) 来管理元素的优先级. 这使得最高优先级的元素总是在队列的前面.
  • 默认比较器: 默认情况下, priority_queue 使用元素类型的比较操作符来确定元素的优先级. 对于基本数据类型, 最大堆会将最大元素放在前面.

stack

stack 是一种容器适配器, 用于实现栈数据结构. 栈是一种具有特定行为的数据结构, 每次弹出操作都会返回最近添加的元素.

stack 具有以下特点:

  • 栈操作: stack 支持栈的基本操作, 包括入栈 (push) 和出栈 (pop). 每次出栈操作都会返回最近入栈的元素.
  • 基于容器实现: stack 的实现基于其他 STL 容器, 通常是 deque. 这意味着 stack 继承了底层容器的性能特点.

array

array 是一种顺序容器, 用于存储固定大小的数组. 与普通的 C 数组不同, C++中的 array 提供了更多的安全性和便利性, 同时也支持许多 STL 算法和操作.

array 具有以下特点:

  • 固定大小: array 的大小在创建时被指定, 并且不能在运行时更改. 这使得 array 适用于那些大小固定的情况.
  • 类型安全: 与原始 C 数组相比, array 提供更好的类型安全性, 防止访问越界和类型不匹配的问题.

关联容器

关联容器用于存储有序的数据集合, 并且提供了快速查找和访问元素的能力. 与顺序容器不同, 关联容器中的元素是按照其键 (key) 来进行排序和组织的.

STL 关联容器包括 set、multiset、map 和 multimap.

这些关联容器使用平衡二叉搜索树 (通常是红黑树) 来实现, 以保持元素的有序性.

无序关联容器

无序关联容器用于存储无序的数据集合, 并且提供了快速查找和访问元素的能力. 与有序关联容器不同, 无序关联容器中的元素不会按照键 (key) 的特定顺序进行排序.

STL 无序关联容器包括 unordered_set、unordered_multiset、unordered_map 和 unordered_multimap.

这些无序关联容器使用哈希表来实现, 以实现快速的查找和插入操作. 无序容器的插入、删除和查找操作在平均情况下具有很好的性能, 但不保证元素的顺序.

分配器

分配器是负责封装堆内存管理的对象, 它们在整个标准库中使用, 特别是 STL 容器使用它们来管理容器内部的所有内存分配.

标准容器使用默认的分配器称为 std::allocator, 例如当你声明一个简单的 vector 对象时, C++ 编译器默认已经使用了内置的 std::allocator:

自定义 allocator 有许多原因:

  • 有些嵌入式平台没有提供默认的 malloc / free 等底层内存管理函数, 而在绝大多数情况下, std::allocator 的 allocate 函数的实现会调用底层的内存分配函数, 所以此时你需要继承 std::allocator 并重写 allocate 函数, 以实现自定义的内存分配逻辑, 而不是直接调用 malloc.
  • 当在 C++ 中实现自己的数据结构时, 需要继承 std::allocator 来自定义其分配器.

分配器通常具备以下基本的成员函数:

  • T* allocate(size_t n): 分配足够的空间来存储 T 的 n 个实例, 并返回指向它的指针.
  • void deallocate(T* p, size_t n): 释放所分配的内存空间.
  • void construct(T* p, Args ... args): 使用 p 指向的 args 参数构造一个对象 (该接口在 C++20 中已被移除).
  • void destroy(T* p): 调用 p 指向的对象的析构函数 (该接口在 C++20 中已被移除).

标准库中的 allocator 还有一些可选成员:

  • 指向类型 T 的指针, 以及指向类型为 T 的常量指针.
  • 类型 T 的引用, 以及类型 T 的常量引用.
  • 自身类型 T 的别名 value_type.
  • 能表达内存分配模型中可分配的最大对象大小的一个无符号整数类型, 能表示内存分配模型中两个指针区别 (difference_type) 的有符号整数类型.
    • 在 C++ 的内存分配中, 通常会有一些限制, 例如单次分配的内存块大小的上限, 这是由操作系统、硬件架构和编译器等因素决定的.
    • 在内存分配模型中, 两个指针之间的差距, 也就是它们在内存中的位置之间的距离, 通常被表示为一个有符号整数, 称为 difference_type.
