虚拟内存

虚拟内存

由于内存存在问题:

  • 空间不足
    • RISC-V 32 提供了一个 32 位的地址空间

      • 理论上来说, 我们的程序可以访问 232 bytes=4 GB2^{32} \ bytes = 4 \ GB232 bytes=4 GB 的内存

        假设计算机中的 RAM 容量为 1 GB, 由于运行程序的时候, 该程序所有的数据都存储在 RAM 中. 由于 RAM 比较小, 所以无法将所有数据从地址空间中映射到 RAM, 所以当程序访问比 RAM 容量大的地址时, 程序会崩溃.

  • 地址空间中的空洞

    • 假设有一个容量为 4 GB 的 RAM, 程序 1 所占空间为 1 GB, 程序 2 所占空间为 2 GB, 程序 3 所占空间为 2 GB

      首先运行程序 1 和程序 2, 即将程序 1 和程序 2 加载到内存中.

      此时剩余空间不够, 所以程序 3 无法加载到内存中. 当程序 1 运行完成, 内存中剩余空间变为 2 GB, 但是程序 3 仍然无法加载到内存中, 因为程序必须连续地存储在内存中.

  • 不能保护程序不受其他程序的影响

    • 每个程序都可以访问任意 32 位地址空间.

    • 程序之间会相互破坏数据

于是便有了虚拟内存.

虚拟内存是假想的内存, 让每个程序认为自己有一个可访问的 32 位地址空间, 虚拟内存获取程序地址, 并将其映射到 RAM 地址或硬盘上.

对内存问题的解决

  • 解决问题一

    当 RAM 满的时候, 可以直接将程序地址空间映射到硬盘上, 当需要使用的数据在硬盘上时, 需要从硬盘上加载数据到 RAM 中, 如果 RAM 没有剩余空间, 可以驱逐 RAM 中的数据, 再将从磁盘中加载的数据放入 RAM 中.

    由于从硬盘中加载数据到 RAM 中的代价很大, 所以当 RAM 满、不得不从硬盘中获取所需的数据时, 对性能影响很大.

  • 解决问题二

    每个程序有自己的 Map, Map 可以将程序数据映射到 RAM 的任何位置.

  • 解决问题三

    程序 1 和程序 2 的地址通过 Map 映射到 RAM 中, 由于两个 Map 不会将地址映射到 RAM 中相同的位置, 所以两个程序无法访问对方的数据.

    由于不同程序之间可以共享部分数据 (例如 库), 所以虚拟内存能够在保护私有数据的同时共享数据

虚拟内存有两个地址

  • 虚拟地址 (Virtual Address, VA)
    • 程序所看到的地址.
    • 例如 lw t0, 1024(x0), 此处的地址为虚拟地址.
    • 虚拟地址会被放到 Map 中, 然后 Map 会给出对应的物理地址 (数据实际存储的位置).
    • 在 RISC-V 32 中, 我们可以访问 0 ∼\sim∼ 232−12^{32} - 1232−1 的地址.
  • 物理地址 (Physical Address, PA)
    • 计算机中的物理 RAM.
    • 地址空间的大小由你的计算机有多少 RAM 所决定.
    • 例如, 假设你有一个 2 GB 的 RAM, 那么你可以访问 0 ∼\sim∼ 231−12^{31} - 1231−1 的地址 (2 GB = 2312^{31}231 byte).

转换

  1. 程序执行指定虚拟地址的加载.
  2. 计算机将虚拟地址转换为内存中的物理地址.
  3. 如果物理地址不在内存中, 则操作系统从硬盘中加载数据到内存中.
  4. 计算机使用物理地址从内存中读取数据并返回给程序.

页表

页表是从虚拟地址到物理地址的映射.

如果将每一个虚拟地址都映射到一个物理地址并存储在页表中, 则总共需要存储 23222=230\frac{2^{32}}{2^2} = 2^{30}22232​=230 个条目 (一个 VA 到 PA 的映射).

为了减少页表中的大小, 可以将一定范围内的虚拟地址映射到一定范围内的物理地址

当我们从磁盘加载数据时, 我们必须加载一整页, 而不是一个字, 此时一个条目表示一个页 (一定范围的虚拟地址到一定范围的物理地址的映射).

现在页的大小通常是 4 KB (1024 字), 所以对于 32 位机器, 页表可以存储 232212\frac{2^{32}}{2^{12}}212232​ = 2202^{20}220 个页.

额外页表元数据

  • V, Valid: 1, 则页在 RAM 中且映射有效; 0, 页不在 RAM 中.
  • D, Dirty: 1, RAM 中的页和硬盘中的页的数据不一致; 0, RAM 中的页和硬盘中的页的数据一致.
  • R, Read: 读权限.
  • W, Wrtie: 写权限.
  • X, Execute: 执行权限.

其中 R, W, X 位为保护位, 可以阻止进程访问其他进程中无权限访问的页, 如果无权访问页, 则会导致内存保护错误 (Memory protection fault) 或段错误 (Segmentation fault).

页表初始化时将每个虚拟页码映射到磁盘中的页码, 有效位为 0, 这样才能在发生页错误时找到页在磁盘中的位置.

将虚拟内存和物理内存分页, 通过页表进行对应:

  • 页码 (Page Number)
    • 确定地址在内存中的第几页.
    • 虚拟页码 (Virtual Page Number, VPN): 虚拟内存的分页码.
    • 物理页码 (Physical Page Number, PPN): 物理内存的分页码.
  • 页偏移量 (Page offset)
    • 确定地址在内存中的某一页中的位置.

虚拟地址可以拆成虚拟页码 + 页偏移量, 同样物理地址也可以拆成物理页码 + 页偏移量

由于从虚拟地址转换为物理地址后页偏移量不发生改变, 所以在页表中不需要存储页偏移量, 只需要存储虚拟页码和物理页码:

此时从 VPN 到 PPN 的映射便是页表中的条目.

页错误

当需要的页不在 RAM 中时, 会发生页错误.

处理

  1. 硬件 (CPU) 产生页错误异常.
  2. 硬件跳转到操作系统页错误处理器来修复错误.
  3. 如果 RAM 中没有剩余的空间, 则操作系统选择将 RAM 中的一个页逐出 RAM.
  4. 如果被逐出的页被标记为脏 (Dirty, 该页在 RAM 中的数据和在硬盘中的数据不一致), 则需要将该页写回硬盘.
  5. 更新页表, 将被逐出页表的页中的虚拟页码映射到硬盘.
  6. 将所需的页从硬盘中加载到 RAM 中.
  7. 更新页表.
  8. 跳转回产生页错误的指令.

页驱逐策略

  • 先进先出 (First in and first out, FIFO)
    • 驱逐页表中最旧的页.
    • 只有在访问不在 RAM 中的页时, 才需要更新时间表.
  • 最近最少使用 (Least Recently Used, LRU)
    • 驱逐最长时间未使用的页.
    • 每次访问都需要更新时间表.
  • 随机 (Random)
    • 随机驱逐页.

通常情况下 LRU 更优, 但是 LRU 代价也比较大:

  • 需要在每次访问时更新页表.
  • 拥有许多元数据.
  • 如果我们只能在内存中存储 4 个页, 则随机驱逐可能更优
    • 例如: 访问 0, 1, 2, 3, 4, 0, 1, 2, 3

页表缓存

对于虚拟内存, 每次访问一段数据需要进行两次内存访问:

  1. 由于页表存储在内存中, 所以需要访问内存中的页表来将虚拟地址转换为物理地址.
  2. 得到物理地址后访问内存来得到数据.

由于两次访问内存的代价相对较大, 而第二次对内存的访问无法避免, 于是可以用缓存来加快第一次访问.

页表的缓存也被称为 Translation Lookaside Buffer (TLB).

  • 如果缓存命中, 则直接从缓存中获取对应的物理地址, 并访问 RAM 以获取数据.
  • 如果缓存未命中, 则从页表中加载映射到缓存中 (如果页表表明数据在硬盘中, 则需要先更新页表) , 然后访问 RAM 以获取数据.

为了让 TLB 尽量小而快, TLB 通常是全关联映射的, 通常包含 32 - 128 个条目.

页表缓存刷新 (TLB Flush)

当上下文切换, 即切换进程时, 需要刷新页表缓存.

因为 TLB 中的所有条目是相对于当前执行的进程而言的, 当切换时, 需要使所有的条目无效.

当有了页表缓存后, 无需在页表中存储 VPN, 因为 VPN 可以作为索引来确定对应的物理地址在页表中的位置.

多级页表

由于每个程序都需要一个独立的页表, 且页表存储在 RAM 中, 当运行的程序足够多时, 将占据大量内存空间.

于是便有了多级页表 (Multi-level Page Tables).

对于两级页表, 第一级页表包含第二级页表的物理地址, 第二级页表包含数据的物理地址.

其中第一级页表必须存储在 RAM, 而第二级页表可以存储在 RAM 中, 也可以存储在硬盘中.

因为通过第一级页表来索引第二级页表的位置, 所以第一级页表必须存储在 RAM 中, 为了减小页表在内存中所占空间, 开始将第二级页表存储在硬盘中, 当需要访问某一第二级页表时, 将其加载到内存中.

一级页表的大小等于 4 KB, 则第一级页表中页表条目的数量为 22×21022=210=1024\frac{2^2 \times 2^{10}}{2^2} = 2^{10} = 10242222×210​=210=1024, 即第一级页表占 10 位. 同理第二级页表占 10 位.

缓存与虚拟内存

  • 物理索引/物理标记缓存 (PIPT)

    由于缓存中包含的是物理索引/物理标记, 所以同一个缓存可以被多个进程共享.

    但是在访问缓存之前, 必须先将虚拟地址转换为物理地址, 需要一定的时间代价.

  • 虚拟索引/虚拟标记缓存 (VIVT)

    由于缓存中包含的是虚拟索引/虚拟标记, 所以同一个缓存不能被多个进程共享.

    由于同一个缓存不能被多个进程共享, 所以必须在每次上下文切换的时候刷新缓存.

  • 虚拟索引/物理标记缓存 (VIPT)

    由于物理页偏移量等于虚拟页偏移量, 所以可以将虚拟页偏移量拆成索引和位偏移量来确定数据在缓存中的位置, 并行查找 TLB 和 缓存, 通过比较物理地址的 Tag 和缓存中找到的缓存行的 Tag 是否相等来判断是否命中.

    VIPT 能够使同一个缓存被多个进程共享, 且时间花费小于 PIPT.
