数字表示

数字表示

位、字节、字

计算机中所有低层次的东西都表示为位, 例如数值、文字、图像、语音等 :

位 (bitbitbit): 000 或 111.

字节 (bytesbytesbytes): 888 位等于 111 字节.

字 (wordwordword): z在 323232 位架构的计算机中, 一个字是 323232 位; 在 646464 位架构的计算机中, 一个字就是 646464 位.

为什么用位: 0/10/10/1 在硬件级别上非常容易表示.

用 NNN 位可以表示 2N2^N2N 种事物.

原码

原码用最高位表示符号位, 其他位存放该数的二进制的绝对值,.

例如: 101010101010, 最高位为 111, 表示负数, 其他三位 010010010, 即 0∗22+1∗21+0∗20=20*2^2 + 1*2^1+0*2^0=20∗22+1∗21+0∗20=2, 所以 101010101010 表示十进制数 −2-2−2.

正数之间用原码进行加法不会出错, 而负数和正数相加或负数和负数相加就会出错.

原码特点

  • 直观易懂, 和真值转换容易.
  • 原码中 000 有两种不同的表示形式, 给使用带来不便.
  • 原码加减运算易出错.
  • 一个数加上它的相反数不等于 000.

反码

为解决原码的问题, 反码用一个正数除符号位外按位取反来表示负数.

正数的反码还是等于原码, 负数的反码就是它的原码除符号位外按位取反.

反码虽然解决了一个数和其相反数相加不为 000 的问题, 但在两个负数运算的时候仍然会出错.

反码特点:

  • 和原码一样, 用符号位表示数值的正负, 000 为正, 111 为负.
  • 同样, 在反码中 000 有两种表示方式.
  • 反码的表示范围和原码的表示范围相同.

反码在计算机中往往作为数码变换的中间环节.

补码

正数的补码等于它的原码, 负数的补码等于反码 + 111.

补码转十进制, 除了符号位的权值是负的, 其他位的权值都是正的.

例如: 10112=1∗(−2)3+0∗22+1∗21+1∗20=(−5)101011_2 = 1 * (-2)^3 + 0 * 2^2 + 1 * 2^1 + 1 * 2^0 = (-5)_{10}10112​=1∗(−2)3+0∗22+1∗21+1∗20=(−5)10​

将一个数的补码的所有位 (包括符号位) 翻转后加 111 能够得到它的相反数的补码 ^1b5d5f

补码特点:

  • 符号位仍然表示数值的正负.
  • 000 只有一种表示方式.
  • 能够正常进行加减运算.

移码

移码有个偏置值 (bias), 移码的真实值等于二进制值加上偏置值.

例如:

偏置值 bias = -127.

二进制数 00000000 的真值 = 0 + (- 127) = -127.

二进制数 11111111 的真值 = 282^828 - 1 + (-127) = 128.

二进制数 01111111 的真值 = 272^727 - 1 + (-127) = 0.

所以偏置值将二进制原来能表示的范围 [0, 255] 转化为范围 [-127, 128], 即可以用无符号数来表示有符号数.

偏置能将当前的范围映射到任意的范围 (当然, 这个任意的范围是连续的, 且所包含的元素的个数不发生变化).

不过移码进行运算时会有不便, 当对真值进行加法运算的时候, 需要减去一个偏置值, 因为对两个真值进行加法时, 加了两次偏置值; 同理, 对真值进行减法运算时, 需要加上一个偏置值, 因为对两个真值进行减法时, 两个偏置值抵消了.

例如:

有真值 A 和真值 B,

A + B - 偏置值= (二进制A + 偏置值) + (二进制 B + 偏置值) - 偏置值 = (二进制A + 二进制B) + 偏置值;

A - B + 偏置值 = (二进制A + 偏置值) - (二进制 B + 偏置值) + 偏置值 = (二进制A + 二进制B) + 偏置值.

通过观察发现当符号位的输入进位不等于符号位的输出进位时, 便会发生溢出.

例如:

符号位有输入进位 111, 而没有输出进位, 此时便发生溢出.

符号位没有输入进位, 但有输出进位, 此时便发生溢出.

浮点数

效仿十进制的处理方式:

由于采取固定的二进制的形式来表示浮点数, 用有限的位难以表示很大的范围, 也很难确保表示的数的精度, 所以效仿十进制的科学计数法:

浮点数的常规表示: (−1)S × 1.xxx...x × 2(yyy...y−bias)(-1)^S \ \times \ 1.xxx...x \ \times \ 2^{(yyy...y - bias)}(−1)S × 1.xxx...x × 2(yyy...y−bias).

  • S 表示符号位, 1 为负, 0 为正.
  • x (Significand) 表示小数位, 标准浮点数整数部分默认为 1, 所以无需存储; 非标准浮点数整数部分默认为 0.
  • y (Exponent) 表示指数, 采取移码的形式, 标准浮点数的偏置值 (bias) 一般为 - (2n−1−12^{n - 1} - 12n−1−1), 其中 n 为二进制位数, 因为这样可以正好将无符号数的范围分为正负两部分; 而非标准浮点数的偏置值为 -(2n−1−22^{n - 1} - 22n−1−2).

采用移码来表示指数的原因: 为了方便根据指数的大小对浮点数的大小进行比较.

什么时候采用非标准浮点数:

以单精度浮点数为例, 由于: 当指数 (Exponent) 和小数部分 (Significand) 皆为 0 时, 规定该单精度浮点数为 0. 假设让指数为 0 而小数部分不为 0 的浮点数无定义. 设 a 为浮点数正区间内离 0 最近的数, b 为 a 后一个数, 则 a = 1.0...0×200...01+(−127)1.0...0 \times 2^{00...01 + (-127)}1.0...0×200...01+(−127) = 2−1262^{-126}2−126, b = 1.0...01×200...01+(−127)1.0...01 \times 2^{00...01 + (-127)}1.0...01×200...01+(−127) = 2−126+2−1492^{-126} + 2^{-149}2−126+2−149, 则 a 到 0 的距离为 a - 0 = 2−1262^{-126}2−126, b 到 a 的距离为 b - a = 2−1492^{-149}2−149, 此时 a 和 0 之间会有相当大的空区间:

而为了去掉空区间, 我们选择在指数为 0 而小数部分不为 0 的时候让浮点数为非标准浮点数 (非标准浮点数选择整数部分为 0 且指数部分为 -(2n−1−22^{n - 1} - 22n−1−2) 是为了让该浮点数的值介于 0 ~ a 之间, 这样就能够弥补之前的空区间, 表示小的数).

对于浮点数的计算, 需要注意有时候可能会丢失少量精度.

例如: 对于标准浮点数, 计算 222+2−10=2222^{22} + 2^{-10} = 2^{22}222+2−10=222. 因为标准浮点数表示为 (1+0.x...x)×2Exponent(1 + 0.x...x) \times 2^{Exponent}(1+0.x...x)×2Exponent, 假设有足够的位来保存尾数, 则 222+2−10=(1+2−32)×2222^{22} + 2^{-10} = (1 + 2^{-32}) \times 2^{22}222+2−10=(1+2−32)×222, 即小数至少需要 32 位才能保存最终的结果.

单精度浮点数

在 C 语言中为 float, 占 32 位.

标准单精度浮点数如下:

  • 符号位占 1 位.
  • 指数占 8 位, 偏置 (bias) 为 -127.
  • 小数占 23位.

  • 当指数 (Exponent) 和小数部分 (Significand) 皆为 0 时, 规定该单精度浮点数为 0;
  • 当指数为 255 (28−12^8 - 128−1) 且小数部分为 0 时, 规定该单精度浮点数为无穷大;
  • 当指数为 255 且小数部分不为 0 时, 规定该单精度浮点数为 NaN (Not a number), 即无定义.
  • 当指数为 0 且小数部分不为 0 时, 规定此时的单精度浮点数为非标准浮点数.
  • 当指数为 1-254, 小数部分为任何值, 规定此时的单精度浮点数为标准浮点数

双精度浮点数

在 C 语言中为 double, 占 64 位.

标准双精度浮点数如下:

  • 符号位占 1 位.
  • 指数占 11 位, 偏置 (bias) 为 -1023.
  • 小数占 53 位.

其他与单精度浮点数一致.
