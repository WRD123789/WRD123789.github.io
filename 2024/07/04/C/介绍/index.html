C 介绍

介绍

C 语言允许我们利用架构的底层特性 (内存管理、特殊指令) 并且可移植 (C 编译器普遍适用于所有现有的处理器架构).

C 编译器将 C 程序直接映射到特定于体系结构的机器代码, 处理器直接执行机械代码.

C 语言先将源文件 (.c) 编译为对象文件 (.o), 然后通过连接器将这些目标文件和预置的库文件链接在一起, 产生一个可执行文件. 这样当一个大程序的一部分发生改变时, 不需要重新编译整个程序, 只需要重新编译发生改变的部分.

编译优势:

  • 出色的运行时性能. 编译器会化很多时间为给定的体系结构优化代码.
  • 编译时间合理. 仅仅重新编译修改过的文件

编译劣势:

  • 编译产生的文件 (包括目标文件中的可执行文件) 都是特定于体系结构的, 因此通常需要在每个新系统上重新构建可执行文件.
  • “更改 -> 编译 -> 运行” 迭代周期在开发过程中可能会慢.
