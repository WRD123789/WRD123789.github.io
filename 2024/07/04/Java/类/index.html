类

静态/动态

静态属性、静态方法: 在属性或方法中包含关键字 static, 也称为类属性和类方法, 即可以通过类或实例来调用和访问的方法或属性, 但是静态方法中不能访问动态属性, 因为动态属性是实例的属性.

动态属性、动态方法: 在属性或方法中不包含关键字 static, 也称为实例属性和实例方法, 即只能通过类的实例来调用和访问的方法或属性.

访问修饰符

访问权限主要有以下四种:

  • 默认 (即没有修饰符): 只在自己的包中可见.
  • Public: 对所有类都可见.
  • Private: 只在当前类中可见.
  • Protected: 只在当前包或另一个包中当前类的子类中可见.

  • 第一列表示类本身是否有权利访问由访问修饰符定义的成员.
  • 第二列表示当前类所属的包中的类是否有权利访问由访问修饰符定义的成员.
  • 第三列表示在此包外部声明的该类的子类是否有权访问由访问修饰符定义的成员.
  • 第四列表示所有类是否有权访问有访问修饰符定义的成员.

  • 非静态嵌套类: 可以访问其封闭类的任何成员, 即静态属性、静态方法、动态属性和动态方法, 包括以私有修饰符 Private 修饰的成员.

继承

如果 “A 是 B”, 则称 B 是 A 的上位词, 例如: 贵宾狗、雪橇狗、约克狗都是狗, 所以可以称狗是贵宾狗、雪橇狗、约克狗的上位词.

相反, 如果 “A 是 B”, 则称 A 是 B 的下位词, 例如上例中的贵宾狗、雪橇狗、约克狗都是狗的下位词.

封装

将类的属性和方法结合成一个单独的整体, 隐藏具体的实现细节, 提供对外的接口, 能够减少程序的复杂度, 使代码使用者无需考虑内部实现, 只需要根据提供的接口使用即可.

继承可能打破封装: 如果一个子类依赖于其超类中特定功能的实现细节. 而超类的实现可能随着发行版本的不同而改变, 即使子类的代码没有变化, 也可能导致子类被破坏.
