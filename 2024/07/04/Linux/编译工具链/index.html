编译工具链

编译工具链

在 Linux 环境下, 我们一般使用编译工具链, 又称为软件开发工具包 (SDK, Software Development Kit). Linux 环境下常见的编译工具链有: gcc 和 lang, 我们使用的是 gcc.

编译

gcc 和 g++ 都是 GNU 编译器集合的一部分, 其中 gcc 主要用于编译 C 语言程序, 而 g++ 专门用于编译 C++ 程序. 下面是 gcc 的编译过程:

  1. 预处理. 预处理阶段处理所有的预处理指令, 并生成一个扩展的源代码文件
  2. 编译. 编译阶段将预处理后的源代码转换为汇编代码
  3. 汇编. 汇编阶段将汇编代码转换为机器码, 生成目标文件 (通常以 .o 结尾)
  4. 链接. 链接阶段将目标文件和所需的库文件链接在一起, 生成最终的可执行文件

常用选项

gcc 中常用的选项如下:

选项 含义
-Wall 生成所有警告信息
-O0, -O1, -O2, -O3 编译器的 4 个优化级别, -O0 表示不优化, -O1 是默认值, -O3 的优化级别最高
-g 使编译器在编译的时候产生调试相关的信息
-Dmacro 相当于在文件的开头加上 #define macro
-Dmacro=value 相当于在文件的开头加上 #define macro value
-I<directory> 将用户指定的目录添加到编译器的头文件搜索路径中, 来找到那些不在标准系统路径中的头文件. 搜索路径为: 当前目录 -> 通过 -I 选项指定的目录 -> 标准系统头文件目录 (xx/include)

可以使用 cpp -v 命令查看系统的 include 目录.

条件编译

条件编译是指在编译过程中根据特定条件选择性地编译代码的一部分, 主要涉及以下预处理指令:

  1. #if [#elif] [#else] #endif
  2. #ifdef [#elif] [#else] #endif
  3. #ifndef [#elif] [#else] #endif

示例:

  1. #ifdef 和 #ifndef. 这些指令用于检查某个宏是否已定义
  2. #if 和 #elif. 这些指令用于根据表达式的值进行条件判断
  3. #else 和 #endif. 这些指令与 #if 或 #ifdef 配合使用, 用于定义条件不满足时的代码块

Makefile

Makefile 是一个配置文件, 用于指导 make 工具如何编译和链接程序. 它定义了一组规则, 每个规则指定了一个目标文件如何由依赖文件生成.

并且, Makefile 采用的是 “增量编译”, 也就是说, 它只会编译那些修改过的、新增的源文件; 那些没修改过的源文件, 不会被重新编译.

规则

Makefile 的核心就是规则, 一个规则由三部分组成:

  • 目标 (Target): 要生成的文件或动作的名称, 通常是可执行文件、目标文件或其他文件.
  • 依赖 (Dependencies): 生成目标文件所需的源文件或其他目标文件.
  • 命令 (Commands): 为生成目标文件而执行的操作, 通常是编译或链接命令.

make 执行过程

当我们执行 make 指令时:

  1. 如果没有指定文件名, make 会在当前目录下搜索 “Makefile” / “makefile” 文件; 也可以通过 -f / --file 指定文件.
  2. 如果找到, make 会把 Makefile 文件中的第一个 target 作为要执行的 target.
  3. 如果  target 文件不存在, 或 dependencies 中有个文件的修改时间晚于 target 文件, 则会执行对应的 command
  4. 如果 dependencies 中的文件不存在, 那么 make 会去搜索以该文件为 target 的规则, 并通过该规则生成文件.

伪目标

伪目标是 Makefile 中的一种特殊目标, 它不对应任何实际文件. 这种目标通常用于执行特定的命令或任务, 例如清理生成文件、编译特定模块等. 伪目标通过 .PHONY 关键字来声明. 伪目标的主要作用是避免与实际文件名称冲突, 并确保目标总是被执行.

变量

Makefile 允许使用变量来简化和管理构建过程中的参数, 类似于 C/C++ 中的宏.

变量的名称可以包含字母、数字和下划线, 并且是大小写敏感的.

变量在声明时需要赋初始值. 使用时, 需要给在变量名前加上 $ 符号, 如果变量名包含多个字符, 我们需要使用小括号 () 或大括号 {}.
