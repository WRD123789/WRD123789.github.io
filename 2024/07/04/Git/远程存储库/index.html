Git 远程存储库

远程存储库

运程存储库只是你的仓库在另一台计算机上的拷贝, 你可以通过互联网与这台计算机通信 – 也就是增加或者获取提交记录.

如果你不小心丢失了本地所有数据, 可以通过远程存储库拿回丢失的数据; 运程存储库还方便你和其他人合作编写项目

远程分支

当和远程存储库通信时, 在本地仓库会多一个名为 origin/main 的分支, 这个分支便被称为远程分支.

图中虚线部分即为远程存储库, 注意到提交记录 c1 处多了一个远程分支 o/main (此处为缩写, 实际为 origin/main).

远程分支反映了运程仓库(在你上一次和它通信时)的状态

运程分支命名格式: <remote-repo-name> / <remote-branch-name>.

远程存储库中每个分支在本地存储库上都有符合上述命名格式的远程分支.

当你和远程仓库通信时, Git 默认将远程仓库的名字设置为 origin.

运程分支重要特性: 当你用 git checkout 移动 HEAD 指针指向远程分支时, 会自动进入分离 HEAD 状态, 即 HEAD 指针不会指向远程分支, 而是指向远程分支指向的提交, 这是 Git 为了防止你直接操作远程分支.

当你在本地库中基于远程库中旧的提交 (在你更新本地库时, 你的同事更新了远程库) 进行了一系列修改后, 调用 git push 时不会有任何反应, 因为你的更新是基于远程库的旧提交, 充满了不确定性, 所以 Git 不允许你变更远程库.

拉取请求

Pull requests 发生在软件开发中, 当开发者/贡献者准备将新的代码更改与主项目存储库合并时.

在 Pull requests 期间, 存储库维护人员检查来自开发者/贡献者的代码更新, 来确定是否与主存储库合并, 如果没有 Pull requests , 未编写完成或错误的代码更新过早地和主存储库合并, 会破坏整个项目, 即 Pull requests 能够更好地维护这个项目.

Pull requests 允许开发者在不影响主项目代码或用户看到的内容的情况下创建新特性或消除 bug, 当他们在本地编写和测试代码更改时就无需担心破坏整个产品.

Pull requests 遵循 5 个基本步骤:

  • 首先开发人员创建主存储库的分支, 然后将其克隆到本地计算机上.
  • 在本地计算机上对代码进行必要的更改或是添加.
  • 一旦开发人员完成了新的代码的测试或更改, 就将这些代码推送 (Push) 到主存储库中的分支上.
  • 发出 Pull requests, 将通知主存储库维护人员进行审查:
    • 不需要修改, 批准新代码通过.
    • 需要修改, 发送评价给开发人员.
  • 将分支存储库中新的代码更改与主存储库合并, 该项目将使用新特性或修复程序的 bug, 即对产品进行了更新.

Pull requests 确保在将给定项目的任何新更新与主存储库合并之前, 已经对其进行了彻底的审查和批准.

创建一个 Pull requests 通常包含以下三个元素:

  • 起稿: 简要描述代码更新, 包括更新类型 (是新特性还是 bug 修复), 以及它的源 存储库/分支 和它的目标 存储库/分支
  • 合并: 一旦开发者的 Pull requests 得到存储库维护人员的批准, 就可以进行合并.
  • 更新: 如果存储库维护人员在合并之前对代码更新进行任何更改, 则会向开发人员发出警告, 并提供修改问题的任何批注或反馈. 一旦开发者解决了问题, 可以通过添加新的提交来更新 Pull requests, 以便让存储库维护人员再次进行审查和批准.

远程跟踪

将远程存储库克隆到本地计算机时, Git 会为每个分支在本地计算机中创建一个远程分支 (比如: o/main), 且每个分支都有一个 reomote tracking 属性, 该属性决定了本地分支与远程分支的关联关系, 即本地分支跟踪指定远程分支 (例如 本地分支main跟踪远程分支 o/main), 该关系为本地分支指定了 push 的目的地以及 pull 的合并目标 (例如: pull 操作时, 提交记录会先下载到 o/main 上, 再合并到本地分支 main, 这便是隐含的合并目标; push 操作时, 我们把提交记录从本地分支 main 中推送到远程存储库中的 main 分支中, 同时更新远程分支 o/main, 这便是指定了 push 的目的地).

有两种方式可以指定本地分支的关联关系, 来跟踪远程分支:

  • 使用 checkout -b 关键字:

    可以创建一个名为 [local-branch-name] 的分支, 它跟踪远程分支 [remote-branch-name], 并使 HEAD 指针指向它.
