套接字和标准 I/O

标准 I/O 函数的优点

标准 I/O 函数的两大优点为:

  • 标准 I/O 函数具有良好的移植性. 不仅是 I/O 函数, 所有标准函数都有良好的移植性. 因为为了支持所有操作系统, 这些函数都是按照 ANSI C 标准定义的.

标准 I/O 函数的缺点

标准 I/O 函数同样有缺点:

  • 不容易进行双向通信.
  • 有时可能频繁调用 fflush 函数. fflush 是 C 语言标准库提供的一个函数, 用于刷新文件流的缓冲区来确保数据被及时写入文件:
    • 如果 stream 是空指针, 则 fflush 会刷新所有打开的流.
    • 如果 stream 不是空指针, 那么 fflush 会刷新指定的流. 如果 stream 是输出流, 它将强制将缓冲区的内容写入文件或设备; 如果 stream 是输入流, 它将丢弃缓冲区中的未读数据.
  • 需要以 FILE 结构体指针的形式返回文件描述符. 为了使用标准 I/O 函数, 需要使用 FILE 结构体指针, 而创建套接字时默认返回文件描述符, 因此需要将文件描述符转化为 FILE 指针.
