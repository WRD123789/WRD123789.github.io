多播与广播

多播与广播

多播

多播是一种网络通信模式, 允许一台主机同时向多台主机发送相同的消息. 在多播通信中, 数据包被发送到一个特定的多播组, 而所有希望接收这个组数据的主机都可以加入该组. 只有加入了特定多播组的主机才能接收到组内的消息.

多播的数据传输方式及流量方面的优点

多播的数据传输特点可总结如下:

  • 多播服务器端针对特定多播组, 只发送一份数据.
  • 即使只发送一份数据, 但该组内的所有客户端都会接收到数据.
  • 多播组数可以在 IP 地址范围内任意增加.
  • 加入特定组即可接收发往该多播组的数据.

多播组是 D 类地址 (224.0.0.0 ~ 239.255.255.255), 「加入多播组」可以理解为通过程序完成如下声明: “在 D 类地址中, 我希望接收发往目标 239.234.218.234 的多播数据”.

多播是基于 UDP 完成的, 因此, 多播数据包的格式与 UDP 数据包相同. 只是与一般的 UDP 数据包不同, 向网络传递 1 个多播数据包时, 路由器将复制该数据包并传递到多个主机, 即多播需要借助路由器来完成:

上图表示传输至 AAA 组的多播数据包借助路由器传递到加入 AAA 组的所有主机的过程.

因为多播的这种特性, 多播主要用于 “多媒体数据的实时传输”.

另外, 虽然理论上可以完成多播通信, 但是不少路由器并不支持多播, 或者因为网络阻塞问题而阻断多播. 因此, 为了在不支持多播的路由器上完成多播通信, 也会使用隧道技术, 本章我们只讨论支持多播服务的环境下的编程方法.

路由和 TTL 以及加入组办法

为了传递多播数据包, 必须设置 TTL. TTL (Time to Live, 生存时间) 是决定 “数据包传递距离” 的主要因素. TTL 用整数表示, 并且每经过 1 个路由器就减少 1, 当 TTL 变为 0 时, 该数据包就无法被传递, 只能销毁. 因此 TTL 的值设置过大将影响网络流量. 当然, 设置过小将无法传递到目标.

广播

广播是一种网络通信模式, 其中一台主机向同一网络中的所有其他主机发送相同的消息. 广播传输数据的范围和多播有一定的区别, 多播即使在跨越不同网络的情况下也能传输数据; 广播只能向在同一个网络中的主机传输数据.

广播的实现方法

与多播相同, 广播也是基于 UDP 完成的. 根据传输数据时使用的 IP 地址的形式, 广播分为如下 2 种:

  • 直接广播
  • 本地广播

二者在代码实现上的差别主要在于 IP 地址:

  • 直接广播的 IP 地址中除了网络地址外, 其余主机地址全部设置为 1.
    • 例如: 向网络地址为 192.12.34 的网络中的所有主机广播传输数据时, 可以向 IP 地址 192.12.34.255 传输数据.
  • 本地广播的 IP 地址所有位全部设置为 1, 所以本地广播中使用的 IP 地址固定为 255.255.255.255.
    • 例如: 在网络地址为 192.32.24 的网络中的某台主机要向该网络中的其他主机广播传输数据, 则可以向 IP 地址 255.255.255.255 传输数据.
