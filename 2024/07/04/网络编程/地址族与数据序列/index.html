地址族与数据序列

地址族与数据序列

分配给套接字的 IP 地址和端口号

IP 是 Internet Protocol 的简写, 是分配给计算机用于在网络上收发数据的地址值.

地址族是网络编程中用于标识套接字地址结构的一种分类系统. 地址族定义了套接字地址的格式以及如何在网络上标识主机和网络.

为了使计算机能够连接到网络并收发数据, 必须向其分配 IP 地址.
IP 地址主要分为两类:

  • IPv4 (Internet Protocol version 4) ---- 4 字节地址族
  • IPv6 (Internet Protocol version 6) ---- 16 字节地址族

IPv4 与 IPv6 的差别主要是表现在 IP 地址所用的字节数, 目前通用的地址族为 IPv4.

IPv4 标准的 4 字节 IP 地址分为网络地址和主机地址, 有 A、B、C、D、E 等类型:

网络地址

网络地址是为区分网络而设置的一部分 IP 地址. 假设向 WWW.SEMI.COM 公司传输数据, 该公司内部构建了局域网, 把所有计算机连接起来. 当向 WWW.SEMI.COM 网络传输数据时, 先通过网络地址找到该公司的边缘路由器, 当边缘路由器接收到数据后, 再通过主机地址在公司局域网内部找到目标计算机并将数据传输给它.

路由器和交换机

若想构建网络, 需要一种物理设备完成外网与本网主机之间的数据交换, 这种设备便是路由器或交换机. 它们实际上也是一种计算机, 只不过是为特殊目的而设计运行的, 因此有了别名.所以, 如果在我们使用的计算机上安装适当的软件, 也可以将其用作交换机. 另外, 交换机比路由器功能要简单一些, 而实际用途差别不大.

网络地址分类与主机地址边界

只需通过 IP 地址的第一个字节即可判断网络地址占用的字节数, 因为我们根据 IP 地址的边界区分网络地址:

  • A 类地址的首字节范围: 0~127
  • B 类地址的首字节范围: 128~191
  • C 类地址的首字节范围: 192~223

或

  • A 类地址的首位以 0 开始
  • B 类地址的前 2 位以 10 开始
  • C 类地址的前 3 位以 110 开始

用于区分套接字的端口号

IP 用于区分计算机, 有 IP 地址就能向目标主机传输数据, 但仅凭这些无法传输给最终的应用程序. 端口号就是在同一操作系统内为区分不同套接字而设置的, 因此无法将 1 个端口号分配给不同套接字.

操作系统根据网卡中接收到的数据中包含的端口号, 将该数据传输给相应端口号的套接字:

端口号由 16 位构成, 可分配的端口号范围是 0-65535. 但 0-1023 是知名端口号, 一般分配给特定应用程序, 所以应当分配此范围之外的值.

另外, 虽然端口号不能重复, 但 TCP 套接字和 UDP 套接字不会共用端口号, 所以允许重复.

网络字节序与地址变换

字节序与网络字节序

CPU 在内存中保存/解析数据的方式有两种:

  • 大端序: 高位字节存放到低位地址.

  • 小端序: 高位字节存放到高位地址.
    例如:

由于每种 CPU 的数据保存方式不同, 因此, 代表 CPU 数据保存方式的主机字节序在不同 CPU 中也各不相同.

那么当大端序系统和小端序系统通过网络传输数据的时候可能导致数据混乱, 因此, 在网络传输数据时需要约定统一的方式, 这种约定称为网络字节序, 网络字节序和大端序一致, 即先把数据转化为网络字节序序再进行网络传输, 同样, 计算机接收数据时应该将数据识别为网络字节序.

比特序

比特序, 顾名思义, 一个字节内, 多个 bit 的顺序. 在一般情况下, 比特序的顺序与字节序是保持一致的, 即如果是大端字节序, 则高位 bit 存在低地址, 低位 bit 存在高地址.

既然存在比特序, 为什么不存在比特序转换函数呢?

在网络传输过程中, 由网卡完成比特的发送和接收. 在比特流的传输上, 由发送方主机网卡统一定义为先发 MSB (权重最高的 bit 数据, 对于大端主机来说, 一个字节中权重最高的 bit 在高地址; 对于小端主机来说, 一个字节中权重最高的 bit 在低地址), 而接收方主机接收时先接收到的是 MSB, 再根据网卡所在的主机的比特序存储, 这样就排除了大小端之间比特序的影响.

例如:

  • 大端序主机发送给小端序主机
  • 小端序主机发送给大端序主机

比特的发送、接收顺序对 CPU 和软件来说是不可见的, 网卡会自动完成这种转换, 所以在不同机器之间进行网络通信的时候就不需要担心比特序导致的问题.
