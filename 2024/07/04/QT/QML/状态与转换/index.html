QML 状态与转换

状态与转换

用户界面的某些部分通常可以用状态来描述. 状态定义了一组属性变化, 并可由特定条件触发.

此外, 这些状态切换还可以附加一个转换, 该转换定义了这些变化的动画效果或者应该使用的附加操作.

状态

创建状态

上述代码中用到的 PropertyChanges 是状态改变类型, 主要的四种状态改变类型如下:

  • PropertyChanges: 状态改变时改变对象的属性, 如颜色或大小.
  • ParentChange: 在状态改变时改变元素的父对象, 它的核心属性是 parent, 表示该元素新的父对象.
  • AnchorChanges: 在状态改变时改变元素的相对位置, 它的核心属性是 anchors, 这是一个属性组, 包含了 left、right、top、bottom 等子属性, 这些属性定义了目标元素在不同状态下的位置锚点.
  • StateChangeScript: 在状态改变时执行一些脚本或代码, 它的核心属性是 name 和 script, name 是脚本的名称, script 是具体要执行的脚本.

扩展状态

转换

每个元素都可以添加一系列转换. 转换由状态变化执行.

在 QML 中, 转换由 Transition 元素定义, 它有一个起始状态 from 和一个结束状态 to, 以及一系列动画效果. 可以使用 from: 和 to: 属性定义在哪种状态变化中应用特定的转换. 这两个属性就像是一个过滤器, 当过滤器为真时, 转换将被应用. 你也可以使用通配符 “*” 来表示 “任何状态”.

要为元素指定转换效果, 需要使用 transitions 属性, 它接收一个 list<Transition> 列表, 定义了当项目改变状态时所应用的转换效果.
