QT 多线程

多线程

在进行桌面应用程序开发的时候, 假设应用程序在某些情况下需要处理比较复杂的逻辑, 如果只有一个线程去处理, 就会导致窗口卡顿, 无法处理用户的相关操作. 这种情况下就需要使用多线程, 其中一个线程处理窗口事件, 其他线程进行逻辑运算, 多个线程各司其职, 不仅可以提高用户体验还可以提升程序的执行效率.

在 QT 中, 处于 main() 函数中、并且由QCoreApplication::exec() 创建开启的事件循环称为主循环, 主循环所在的线程称为主线程, 也叫 GUI 线程, 负责窗口事件处理或者窗口控件数据的更新; 子线程负责后台的业务逻辑处理, 子线程中不能对窗口对象做任何操作, 这些事情需要交给窗口线程处理; 主线程和子线程之间如果要进行数据的传递, 需要使用 QT 中的信号槽机制.

使用

QThread 的第一种使用方式如下:

  1. 需要创建一个线程类的子类, 让其继承 QT 中的线程类 QThread
  2. 重写父类的 run 方法, 在该函数内部编写子线程要处理的具体的业务流程
  3. 在主线程中创建子线程对象
  4. 启动子线程

当子线程被创建出来之后, 父子线程之间的通信可以通过信号槽的方式.

例如: 假如要实现一个能计数且能实时显示在窗口的程序, 如果只有一个线程, 则会发现数字不能在窗口中实时显示, 此时用户通过鼠标拖拽窗口会导致无响应的情况, 可以用多线程来优化. 设计思路为: 点击 “开始” 按钮后创建子线程进行计数, 并通过信号槽将数据传递给主线程, 再由主线程将数据更新到窗口中.

QThread 的第二种使用方式如下:

  1. 创建一个新的类, 让这个类从 QObject 派生
  2. 在这个类中添加一个 public 的成员函数, 函数体中的就是我们要在子线程中执行的业务逻辑
  3. 在主线程中创建一个 QThread 对象, 这就是子线程的对象
  4. 在主线程中创建工作的类对象 (千万不要给创建的对象指定父对象, 否则会导致下一步的失败)
  5. 将上一步创建的工作的类对象移动到创建的子线程对象中, 需要调用 QObject 类提供的 moveToThread 方法
  6. 通过调用 start 函数启动子线程, 但是移动到线程中的对象并没有开始工作, 必须调用对象的工作函数才能让它开始执行.

线程的事件循环用于为线程中的所有 QObjects 对象分发事件; 默认情况下, 这些对象包括线程中创建的所有对象, 或者是在别处创建完成后被移动到该线程的对象. 一个对象所依附的线程是指它所在的那个线程, 而一个对象在哪个线程中被实例化, 该对象就依附于哪个线程.

我们可以使用线程安全的 QCoreApplication::postEvent() 函数向一个对象发送事件. 它将把事件加入到对象所在的线程的事件队列中. 当然, 如果这个线程没有启动事件循环, 这个事件也不会被派发.

在 Qt 中, 多线程环境下一般使用 deleteLater 而不是 delete, 因为 deleteLater 不会立刻删除对象, 它会将删除操作放入事件循环, 当事件循环下一次运行时, 才会真正地删除对象. 如果在处理事件的过程中删除了对象, 那么之后还未处理的事件就会发送到一个已经不存在的对象中, 这会导致程序崩溃.

QObject 的线程依附性是可以改变的, 方法是调用 QObject::moveToThread() 函数. 该函数会改变一个对象及其所有子对象的线程依附性. 由于 QObject 不是线程安全的, 所以我们只能在该对象所在线程上调用这个函数. 另外, Qt 要求 QObject 的所有子对象都必须和其父对象在同一线程, 这意味着:

  • 不能对有父对象 (parent 属性) 的对象使用 QObject::moveToThread() 函数.
  • 不能在 QThread 的子类中以这个对象本身作为父对象创建对象, 例如: 此时 obj 在 run 函数中创建, 依附于子线程, 而它的父对象不依附于该子线程, 导致错误.

资源释放

子线程对象在完成任务后, 需要释放它们所占用的资源, 有两种方式释放它们所占用的资源:

  1. 创建子线程对象时, 指定它们的父对象, 然后通过 QT 的内存回收机制自动回收
  2. 通过信号槽来完成子线程对象资源的释放

线程池

我们使用线程的时候就去创建一个线程, 这样实现起来非常简便, 但是就会有一个问题: 如果并发的线程数量很多, 并且每个线程都是执行一个时间很短的任务就结束了, 这样频繁创建线程就会大大降低系统的效率, 因为频繁创建线程和销毁线程需要时间.

线程池是一种管理和复用线程的机制, 用于提高多线程应用程序的性能和效率. 在多线程环境中, 创建和销毁线程是一项开销较大的操作, 而线程池的目的是在程序启动时预先创建一定数量的线程, 并将它们保存在池中, 随后根据需要重复利用这些线程, 而不是频繁地创建和销毁线程.

线程池的主要组成部分包括:

  1. 任务队列: 用于存储需要被线程池执行的任务. 通过线程池提供的 API 函数, 可以将一个待处理的任务添加到任务队列或从任务队列中删除一个任务.
  2. 线程集合: 由线程池管理器创建的一组线程. 这些线程会从任务队列中获取任务并执行. 如果任务队列为空, 线程将会被阻塞; 如果阻塞之后有了新的任务, 线程的阻塞状态将会被解除.
  3. 线程池管理器: 周期性地对任务队列中的任务数量以及处于忙状态的线程个数进行检测. 当任务过多的时候, 可以适当的创建一些新的线程; 当任务过少的时候, 可以适当的销毁一些线程.

QRunnable

在 QT 中使用线程池需要先创建任务, 添加到线程池中的每一个任务都需要是一个 QRunnable 类型, 因此在程序中需要创建子类继承 QRunnable 这个类, 然后重写 run 方法, 在这个函数中编写要在线程池中执行的任务, 并将这个子类对象传递给线程池, 这样任务就可以被线程池中的某个线程处理.

QThreadPool

QT 中的 QThreadPool 类管理了一组 QThreads, 里边还维护了一个任务队列. QThreadPool 管理和回收各个 QThread 对象. 每个 QT 应用程序都有一个全局 QThreadPool 对象, 可以通过调用 globalInstance 来访问它. 也可以单独创建一个 QThreadPool 对象使用.
