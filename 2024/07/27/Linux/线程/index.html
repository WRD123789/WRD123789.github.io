线程

线程

一个进程在执行时, 会在 CPU 的寄存器中保存一些必要的信息, 如堆栈、代码段等, 这些状态称为上下文. 上下文切换涉及大量寄存器、内存的读取和写入. 此外, 由于 Linux 操作系统支持虚拟内存, 所以每个用户进程都拥有自己独立的地址空间, 此时每个进程都有独立的页目录表. 因此, 进程切换时需要经过两步:

  1. 切换页目录表, 以支持新的地址空间.
  2. 进入内核态, 切换硬件上下文 (如程序计数器 PC、CPU 寄存器、栈指针).

为了减小切换进程的开销, 便产生了线程, 也称为轻量级进程. 我们将一个进程分解成多个线程, 此时每个线程是进程内的一个执行单元, 线程之间并发执行, 这样线程就取代进程成为 CPU 时间片分配和调度的最小单位. 每个线程拥有自己独立的栈和寄存器, 但与同一进程的其他线程共享内存、文件描述符和其他资源.

虽然各个线程的栈是独立的, 但是这些栈位于同一个地址空间中, 因此一个线程可以直接访问另一个线程的栈, 可见同一个进程的多个线程间通信比较方便. 另外, 由于同一个进程的多个线程共享同一个地址空间, 所以同一个进程内的线程切换时不需要切换页目录表, 线程切换时也只需要切换栈、PC 指针以及其他少量控制信息. 因此, 切换线程的开销要小于切换进程的开销.

用户级线程和内核级线程

用户级线程是完全由用户空间的线程库管理的线程, 操作系统内核对其一无所知. 用户级线程的创建、调度和销毁完全由用户空间的运行时库 (也称为管理线程, 本质上是用户空间的一个调度器) 管理, 不需要内核干预. 由于不需要内核态的切换, 因此用户级线程的上下文切换速度非常快. 用户也可以根据需要实现自己的调度算法, 实现特定需求的优化. 如果一个用户级线程执行一个阻塞系统调用, 那么整个进程都会被阻塞.

内核级线程是由操作系统内核直接管理的线程, 每个线程都是内核中的一个调度实体. 内核级线程由操作系统内核管理, 创建、调度和销毁都需要内核参与, 因此上下文切换时开销较大. 由于受内核控制, 因此, 一个内核级线程的阻塞不会影响同一进程中的其他线程, 此时内核可以选择调度其他线程.

有些系统 (如许多现代操作系统) 采用了用户级线程和内核级线程的混合模型, 以便同时利用两者的优点. 这种模型下, 用户级线程库会将用户级线程映射到一个或多个内核级线程上, 这样既可以实现快速的上下文切换和灵活的用户级调度, 又能充分利用多处理器系统的能力. 如下图所示:

pthread_exit 函数用于在线程完成其工作时安全地终止线程. 它允许线程的资源被正确地释放, 并且可以返回一个退出状态给其他线程. 该函数的原型如下:

参数:

  • retval: 指向任意类型的数据的指针, 表示线程的退出状态. 这个值可以在其他线程通过 pthread_join 函数捕获.

当线程调用 pthread_exit 时, 它会执行所有通过 pthread_cleanup_push 注册的清理处理程序, 这些清理处理程序会按照注册时相反的顺序被执行.

需要注意的是, 如果主线程 (即调用 main 函数的线程) 调用 pthread_exit, 那么进程不会终止, 直到所有的线程都终止.

通过 pthread_exit 退出线程和通过 return 简单地从线程函数返回存在一些区别:

  • 调用 pthread_exit 将明确地表明线程的终止; 通过 return 从线程函数返回相当于隐式地调用 pthread_exit, 并将返回值作为退出状态, 其他线程同样可以通过 pthread_join 获取该线程的退出状态.
  • 对于复杂的多线程应用程序, 显式调用 pthread_exit 可以更明确地管理线程的生命周期和资源清理; 通过 return 从线程函数返回时, 即使注册了清理处理程序, 也不会执行注册的清理处理程序, return 一般用于不需要复杂资源管理的简单线程函数.

pthread_cancel 函数用于请求取消一个线程的执行. 该函数的原型如下:

参数:

  • thread: 要取消的线程的标识符, 通常是通过 pthread_create 创建线程时返回的 pthread_t 对象.

pthread_cancel 函数会向指定的线程发送一个取消请求, 默认情况下, 被请求取消的线程会在到达下一个取消点时响应取消请求. 取消点是线程在执行过程中检查是否有取消请求的特定位置, 通常有 pthread_testcancel、阻塞的系统调用 (如 read、write 等) 以及 pthread_cond_wait、pthread_cond_timedwait 等等待条件变量的操作, 而锁相关的操作都不是取消点.

每个线程都有一个取消状态和取消类型:

  • 取消状态: 决定线程是否响应取消请求. 可以是 PTHREAD_CANCEL_ENABLE (默认，表示线程可以响应取消) 或 PTHREAD_CANCEL_DISABLE (表示线程忽略取消请求) .
  • 取消类型: 决定线程何时响应取消请求. 可以是 PTHREAD_CANCEL_DEFERRED (默认，表示线程在下一个取消点响应) 或 PTHREAD_CANCEL_ASYNCHRONOUS (表示线程立即响应取消) .

线程资源清理

一般来说, 线程终止有两种情况: 正常终止和非正常终止. 线程主动调用 pthread_exit 或在线程函数中使用 return 都将使线程正常退出, 这是可预见的退出方式; 非正常终止是线程在其他线程的干预下, 或者由于自身运行出错而退出, 这种退出方式是不可预见的.

不论是可预见的线程终止还是异常终止, 都应该保证线程终止时能顺利的释放掉自己所占用的资源. 例如, 对于资源独占锁的使用: 线程为了访问临界资源而为其添加锁, 但在访问过程中该线程被其他线程取消, 如果线程处于响应取消状态, 且采用异步方式响应, 则该临界资源将永远处于锁定状态得不到释放.

为此, POSIX 线程库提供了 pthread_cleanup_push 和 pthread_cleanup_pop 函数用于在线程取消 (或以其他方式退出) 时执行清理操作. 这两个函数成对使用, 以确保在线程退出时释放资源或执行其他必要的清理工作. 这两个函数的原型如下:

参数:

  • routine: 清理处理程序函数的指针. 这个函数将在清理时被调用.
  • arg: 传递给清理处理程序函数的参数.
  • execute: 一个整数值. 如果非零, 表示立即执行清理处理程序; 如果为零, 仅注销清理处理程序而不执行清理处理程序.

在执行 pthread_cleanup_push 到 pthread_cleanup_pop 之间的程序段时, 如果发生终止动作 (如调用 pthread_exit 和 pthread_cancel), 都将执行 pthread_cleanup_push 所指定的清理函数.

pthread_cleanup_push 和 pthread_cleanup_pop 采用先入后出的栈结构管理, 清理函数在调用 pthread_cleanup_push 时被压入清理函数栈中, 多次对 pthread_cleanup_push 的调用将在清理函数栈中形成一个函数链, 在执行该函数链时按照压栈的相反顺序弹出.

线程的同步与互斥

互斥锁的基本使用

在多线程编程中, 用来控制共享资源的最简单有效的机制是互斥锁. 互斥锁的数据类型是 pthread_mutex_t, 其本质是一个全局的标志位, 线程可以对其进行原子地测试并修改, 即所谓的加锁. 当一个线程持有锁时, 其余线程再尝试加锁时 (包括自己再次加锁), 会使自己陷入阻塞状态, 直到锁被其持有线程解锁才会被唤醒.

条件变量

理论上来说, 利用互斥锁可以解决所有的同步问题, 但是生产实践中往往会出现这样的问题: 一个线程能够执行取决于一个共享资源的具体数值, 而该共享资源的数值会随着程序的运行不断地变化, 如果只使用互斥锁来解决问题, 就会出现大量的、重复的 “加锁 - 检查条件不满足 - 解锁” 的行为, 此时, 不满足条件的线程会经常尝试去占用 CPU 资源, 上下文切换也会比较频繁.

在 Linux 中, 条件变量是用于多线程编程的一种同步机制, 它可以使线程在等待某个条件成立时被阻塞, 当条件成立时, 另一个线程可以通知被阻塞的线程继续执行. 条件变量通常与互斥锁一起使用, 以确保对共享资源的访问是线程安全的.

线程安全性和可重入性

线程安全性

线程安全的函数指的是在多线程环境下调用时, 不会引起竞争条件或数据不一致的情况. 因此, 多个线程可以同时调用这个函数, 而不会导致意外的行为或错误.

线程安全的函数通常通过以下几种方式实现:

  1. 不使用全局或静态变量: 全局和静态变量在多个线程间共享, 如果没有适当的同步机制, 会导致数据竞争.
  2. 使用局部变量: 局部变量存储在线程的栈上, 每个线程都有自己的栈, 互不干扰.
  3. 使用同步机制: 如互斥锁 (mutex) 、信号量 (semaphore) 等, 来保护共享资源.

可重入

可重入的函数是指在执行过程中可以被打断, 并且在被打断后可以再次安全地进入该函数, 且不会引起任何错误或数据不一致的情况. 可重入函数通常是线程安全的, 但并非所有线程安全的函数都是可重入的.

可重入函数的特性包括:

  1. 不依赖于全局和静态数据, 或仅依赖于可重入函数自己管理的全局数据.
  2. 不返回静态或全局数据的指针.
  3. 仅使用局部变量或传递进来的参数.
  4. 不调用不可重入的函数.

一个比较典型的不可重入函数的例子是 malloc. malloc 函数会修改静态数据, 为了保证线程安全性, malloc 函数的实现中会存在加锁和解锁的过程, 如果 malloc 在执行到加锁之后、解锁之前时, 接收到信号, 那么线程会去执行信号处理函数, 假如该信号处理函数中再次调用 malloc 函数, 此时就会导致死锁.
