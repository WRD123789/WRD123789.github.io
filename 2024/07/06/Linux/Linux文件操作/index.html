Linux 文件操作

Linux 文件操作

在 UNIX/Linux 中, 所有满足速度较慢、容量较大或可持久化存储中的任意一个特征的数据集合都可以称为文件.

根据文件存在的形式, 文件分为普通文件、目录文件、链接文件和设备文件等类型:

  • 普通文件: 也称磁盘文件, 并且能够进行随机的数据存储.
  • 管道: 是一个从一端发送数据, 另一端接收数据的数据通道.
  • 目录: 包含了保存在目录中文件列表的简单文件.
  • 设备: 该类型的文件提供了大多数物理设备的接口. 它又可以分为两种类型: 字符设备和块设备. 字符设备一次只能读出和写入一个字节的数据, 如终端、打印机、声卡、鼠标; 块设备必须以一定大小的块来读出或者写入数据, 块设备包括 CD-ROM、RAM 驱动器和磁盘驱动器等. 字符设备一般用来传输数据, 块设备一般用来存储数据.
  • 链接: 类似 Windows 的快捷方式.

基于文件描述符的文件操作

之前所讨论的文件操作都利用了数据结构 FILE, FILE 是文件流, 也称为用户态文件缓冲区, 所以与 FILE 类型相关的文件操作称为带缓冲的 I/O.

POSIX 标准支持另一类无缓冲的 I/O, 即通过文件描述符进行的文件操作.

针对每个进程, 内核都为其维护了打开文件 (也称为文件对象 filestruct) 的文件描述符表. 该表的每一条目都记录了单个文件描述符的相关信息, 如下所示:

  • 控制文件描述符操作的一组标志 (如 close-on-exec 标志).
  • 对打开文件句柄的引用.

内核对所有打开的文件维护有一个系统级的描述表格 (也称作打开文件表), 并将表中各条目称为打开文件句柄. 一个打开文件句柄存储了与一个打开文件相关的全部信息, 如下所示:

  • 当前文件偏移量.
  • 打开文件时所使用的状态标志.
  • 文件访问模式.
  • 与信号驱动 I/O 相关的设置.
  • 对该文件 i-node 对象的引用.

每个文件系统都会为驻留其上的所有文件建立一个 i-node 表, 一个文件的 i-node 信息如下:

  • 文件类型和访问权限.
  • 一个指针, 指向该文件所持有的锁的列表.
  • 文件的各种属性, 包括文件大小以及与不同类型操作相关的时间戳.

磁盘上的 i-node 记录了文件的固有属性, 例如: 文件类型、访问权限和时间戳. 访问一个文件时, 会在内存中为 i-node 创建一个副本, 其中记录了引用该 i-node 的打开文件句柄数量以及该 i-node 所在设备的主、从设备号, 还包括一些打开文件时与文件相关的临时属性, 如: 文件锁.

文件映射

使用 mmap 可以将文件的一部分或全部内容映射到进程的地址空间 (用户态的内存), 使得文件的内容可以像内存一样进行访问:

参数:

  • addr: 指定映射区域的起始地址, 通常为 NULL, 表示由内核选择地址.
  • length: 映射区域的长度 (字节数).
  • prot: 内存保护标志, 控制映射区域的访问权限 (如 PROT_READ、PROT_WRITE、PROT_EXEC、PROT_NONE).
  • flags: 映射选项, 控制映射区域的特性 (如 MAP_SHARED、MAP_PRIVATE、MAP_ANONYMOUS).
    • MAP_SHARED 表示共享, 方便多个进程共享同一个文件的映射区域.
    • MAP_ANONYMOUS 表示匿名映射, 即该映射区域不与任何文件关联, 通常用于创建共享内存.
  • fd: 文件描述符, 表示要映射的文件. 如果是匿名映射, 则应为 -1.
  • offset: 文件的偏移量, 表示从文件的哪个位置开始映射 (mmap 的映射区域大小和偏移量必须是系统虚拟内存页大小的整数倍).

返回值:

  • 成功时返回映射区域的起始地址.
  • 失败时返回 MAP_FAILED.

dup2() 系统调用会为 oldfd 参数所指定的文件描述符创建副本, 其编号由 newfd 参数指定. 如果 newfd 参数指定的文件描述符已经打开, 那么 dup2() 首先会将其关闭. (dup2() 调用会忽略 newfd 关闭期间出现的任何错误. 因此, 更安全的做法是: 在调用 dup2() 之前, 先判断 newfd 是否已经打开, 若打开则应显式调用 close() 将其关闭.

如果 oldfd 并非有效的文件描述符, 那么 dup2() 调用将失败并返回错误 EBADF; 如果 oldfd 有效, 且与 newfd 值相等, 那么 dup2() 将直接返回结果.

I/O 多路复用是一种在单个线程或进程中同时监视多个文件描述符以提高程序并发处理能力的方法. 通过I/O多路复用, 程序可以在一个线程中同时处理多个 I/O 操作, 从而避免了为每个 I/O 操作都创建一个线程或进程的开销.

select

select 函数是 Linux 中用于 I/O 多路复用的经典方法之一, 它允许程序在一个线程中同时监视多个文件描述符, 以检测它们是否准备好进行读、写或有异常条件发生. 该函数的原型如下:

参数:

  • nfds: 一个整数值, 表示待监视的文件描述符集合中最大文件描述符值加 1.
  • readfds: 指向 fd_set 结构的指针, 表示需要监视可读事件的文件描述符集合. 如果不需要监视可读事件, 可以传递 NULL.
  • writefds: 指向 fd_set 结构的指针, 表示需要监视可写事件的文件描述符集合. 如果不需要监视可写事件, 可以传递 NULL.
  • exceptfds: 指向 fd_set 结构的指针, 表示需要监视异常事件的文件描述符集合. 如果不需要监视可写事件, 可以传递 NULL.
  • timeout: 指向 timeval 结构的指针, 表示等待文件描述符事件发生的超时时间. 如果设为 NULL, 则 select 会无限期地等待; 如果设置超时时间为 {0, 0}, 则 select 会立即返回, 用于非阻塞检查.

返回值:

  • 成功时返回就绪文件描述符的总数.
  • 失败时返回 -1, 并设置 errno 以指示错误类型.

上文中所说的 fd_set 实际上是一个位图, 每一个文件描述符对应一个位, 通过位操作来检查和设置文件描述符的状态.

可以使用下面的宏来操作 fd_set:

  • FD_ZERO(fd_set *set): 初始化文件描述符集合, 将集合中的所有位都置为 0.
  • FD_SET(int fd, fd_set *set): 将文件描述符 fd 加入集合 set 中.
  • FD_CLR(int fd, fd_set *set): 从集合 set 中移除文件描述符 fd.
  • FD_ISSET(int fd, fd_set *set): 检查文件描述符 fd 是否在集合 set 中.
