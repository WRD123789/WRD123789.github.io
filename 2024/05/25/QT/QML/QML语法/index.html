QML 语法

QtQuick 是 Qt 的一个模块, 它提供了一种高级的、易于使用的方式来创建动态和流畅的用户界面. 它将用户界面分解为更小的元素, 这些元素可以组合成组件.

QtQuick 允许创建元素的层次结构: 子元素从父元素中继承坐标系统, 它的坐标总是相对于它的父元素的坐标系统.

QML 是 QtQuick 的核心组成部分, 它是一种声明式语言, 用于描述对象之间的关系.

  • 每一个 QML 文档都需要一个根元素.
  • 每个元素先声明类型 (如上例中的 Image、Text), 再用 {} 包含属性, 属性以键值对的形式声明.
  • 可以通过 id 访问 QML 文档中的任意元素.
  • 在 QML 中, 元素可以嵌套, 子元素可以通过 parent 访问它的父元素.

属性

在 QML 中, 元素是由它们的元素名来声明的, 但是是通过它们的属性或者创建自定义属性来定义的. 属性是一个简单的键值对, 如下所示:

  • id 是一个特殊的类属性值, 用于引用 QML 文档中的元素. id 不是字符串类型, 而是一个标识符和 QML 语法的一部分. id 在文档中必须是唯一的, 不能被重新赋值, 也不能被查询.
  • 可以根据属性的类型为其赋值, 未被赋值的属性会根据属性的类型提供一个默认值.
  • 一个属性可以依赖于一个或多个其他属性, 这被称为绑定. 一个绑定属性在其依赖属性更改时更新. 如该例中的 text 属性依赖于 int 类型的自定义属性 times, int 类型的值会自动转换为字符串类型, 当 times 属性更改时会同步更新 text 属性.
  • 可以通过 property 限定符向元素添加新属性, 后跟类型、名称和可选的初始值 (property <type> <name>: <value>), 如果没有提供初始值, 则会根据属性的类型提供默认值.
  • alias 关键字用于创建属性别名 (property alias <name>: <reference>), 使得一个对象的内部属性可以在外部被访问和修改.

绑定

在 QML 中, 绑定 (:) 是一种保持在绑定生命周期内始终为真的协议. 换句话说, 当绑定的源属性发生变化时, 绑定目标的属性会自动更新以反映这个变化. 而 JavaScript 的赋值 (=) 操作只是一次性的值传递. 也就是说, 当你对一个变量进行赋值操作时, 你只是把一个值赋给了这个变量, 之后即使这个值发生了变化, 也不会影响到这个变量.
