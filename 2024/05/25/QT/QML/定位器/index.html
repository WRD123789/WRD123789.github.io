定位器

有许多 QML 元素用于定位元素. 这些元素被称为定位器, Qt Quick 模块定义了以下定位器: Row, Column, Grid 和 Flow.

Column 元素能将所有子元素排成一列, 通过 spacing 属性设置每个子元素之间的距离. 如果 Column 的高度不足以容纳所有的子项, 那么超出的子项会被隐藏或者被截断.

Row 元素能够将所有子项排列成行, 通过 layoutDirection 属性决定从左到右排列或是从右到左排列. 同样通过spacing 属性设置每个子元素之间的距离. 如果 Row 的宽度不足以容纳所有的子项, 那么超出的子项会被隐藏或者被截断.

Grid 元素能够将所有子元素排列到网格中. 通过设置 rows 和 columns 属性, 可以限制行或列的数量. 如果不设置这两个属性中的任何一个, 另一个将根据子元素的数量计算得出. 例如, 将行设置为 3 并添加 6 个子元素, 就会产生 2 列. flow 和 layoutDirection 属性用于控制元素添加到网格中的顺序, 而 spacing 属性则控制子元素之间的间隔.

Flow 元素可以将所有子元素按照预定的顺序排列在一个或多个行 (或列) 中, 通过 flow 和 layoutDirection 控制子元素的排列方向. Flow 元素默认的排列顺序是从左到右, 从上到下. 当 Flow 的宽度 (或高度) 不足以容纳所有子项时, 它会自动将子项移动到新的一行 (或列).

定位器经常使用的元素是 Repeater, 它可以用一个模型和一个代理来创建大量类似的元素. 它的主要属性如下:

  • model: 定义了要重复的元素的数量. 这可以是任何可用的模型, 例如一个整数、一个列表或一个自定义数据模型.
  • delegate: 定义了每个元素的具体内容. 这可以是任意的 QML 类型.
  • count: 返回 Repeater 创建的元素数量.
  • index: Repeater 的内置属性, 它表示当前循环的索引值. 当你在 delegate 属性中创建一个元素时, 这个元素就能直接访问到 index 属性. 这是因为 delegate 属性的内容会在每次循环中被创建一次, 而在每次创建时, 当前的索引值都会被注入到 delegate 中.
