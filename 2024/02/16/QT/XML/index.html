XML

XML (eXtensible Markup Language) 是一种通用的文本格式, 被广泛运用于数据交换和数据存储 (虽然近年来 JSON 盛行, 大有取代 XML 的趋势, 但是对于一些已有系统和架构, 比如 WebService, 由于历史原因, 仍旧会继续使用 XML.

针对 XML 的通用处理, Qt4 提供了 QtXml 模块; 针对 XML 文档的 Schema 验证以及 XPath、XQuery 和 XSLT, Qt4 和 Qt5 则提供了 QtXmlPatterns 模块.

Qt 提供了三种读取 XML 文档的方法:

  • QXmlStreamReader: 一种以快速的基于流的方式访问良格式 XML 文档, 特别适合于实现一次解析器 (所谓 “一次解析器”, 可以理解成我们只需读取文档一次, 然后像一个遍历器从头到尾一次性处理 XML 文档, 期间不会有反复的情况).
  • DOM (Document Object Model): DOM 解析器将整个 XML 文档加载到内存中, 并构建一个树形结构, 其中每个节点都代表 XML 中的一个元素或节点. 适用于对整个文档进行多次操作或复杂的数据处理任务. DOM 方式带来的问题是需要一次性将整个 XML 文档读入内存, 因此会占用很大内存.
  • SAX (Simple API for XML): SAX 解析器是一种基于事件驱动的解析器, 它逐行读取 XML 文档并触发一系列事件, 如元素的开始和结束、文本内容等. SAX 不需要将整个文档加载到内存中, 因此在处理大型文档时占用的内存较少. SAX 不支持随机访问 XML 数据, 而是按顺序处理, 通常用于一次性地读取并处理 XML 数据.

在 Qt4 中, 这三种方法都位于 QtXml 模块中. Qt5 则将 QXmlStreamReader、QXmlStreamWriter 移动到 QtCore 中, QtXml 则标记为 “不再维护”.

这三种方法中, QXmlStreamReader 是 Qt 中最快最方便的读取 XML 的方法. 因为 QXmlStreamReader 使用了递增式的解析器, 适合于在整个 XML 文档中查找给定的标签、读入无法放入内存的大文件以及处理 XML 的自定义数据.

使用 DOM 处理 XML

DOM 是由 W3C 提出的一种处理 XML 文档的标准接口.

与上文中 QXmlStreamReader 不同, DOM 一次性读取整个 XML 文档, 在内存中构造为一棵树 (称为 DOM 树). 我们能够在这棵树上进行移动, , 也可以对这棵树进行修改, 或者是直接将这颗树保存为硬盘上的一个 XML 文件.

使用 SAX 处理 XML

SAX 是一种读取 XML 文档的标准 API, 同 DOM 类似, 并不以语言为区别. Qt 的 SAX 类基于 SAX2 的 Java 实现, 不过具有一些必要的名称上的转换. 相比 DOM, SAX 的实现更底层因而处理起来通常更快. 但是, QXmlStreamReader 类更偏向 Qt 风格的 API, 并且比 SAX 更快, 所以, 现在我们之所以使用 SAX API, 更主要的是为了把 SAX API 引入 Qt. 在我们的项目中, 并不需要真的使用 SAX.

Qt 提供了 QXmlSimpleReader 类, 提供基于 SAX 的 XML 处理. 同前面所说的 DOM 方式类似, 这个类也不会对 XML 文档进行有效性验证. QXmlSimpleReader 可以识别良格式的 XML 文档, 支持 XML 命名空间, 它在解析 XML 文档时以流式方式生成事件, 应用程序可以通过注册事件处理器来响应这些事件. 在解析 XML 文档时, QXmlSimpleReader 会在遇到 XML 文档的不同部分时生成不同的事件, 并调用相应的事件处理器中的虚函数来通知应用程序, 从而使应用程序能够对 XML 内容进行处理.

每出现一个事件, 都会调用对应的回调函数, 这些回调函数定义在称为处理器的接口类中. 上面给出的事件处理函数都定义在 QXmlContentHandler 中. 除 QXmlContentHandler 外, Qt 还有 QXmlEntityResolver、QXmlDTDHandler、QXmlErrorHandler、QXmlDeclHandler 以及 QXmlLexicalHandler 等接口类, 它们都是纯虚类, 分别定义了不同类型的事件处理函数, 其中 QXmlContentHandler 和 QXmlErrorHandler 是最常用的两个.

为简化处理, Qt 提供了一个 QXmlDefaultHandler. 这个类实现了上面所有的接口, 为每个事件处理函数提供了一个空白实现. 因此, 当我们需要实现一个处理器时, 只需要继承这个类, 覆盖我们所关心的几个函数即可.

使用 SAX API 和 QXmlStreamReader 或 DOM API 之间最大的区别是: 使用 SAX API 要求我们必须自己记录当前解析状态, 而 DOM API 和 QXmlStreamReader 会在解析过程中自动构建文档的树形结构或提供一种类似于迭代器的接口, 因此不需要应用程序自己维护解析状态.

  • 在处理事件的过程中, 应用程序可能需要跟踪当前所在的 XML 元素、记录元素的属性、积累字符数据等信息, 这些信息被称为解析的状态. 例如, 当解析器开始解析一个 XML 元素时 (如 <book>), 应用程序需要记录当前所在的元素是 <book>; 当解析器读取到元素的属性时 (如 <book id="123">), 应用程序需要记录 id 属性的值是 “123”.

生成 XML 文档

许多应用程序不仅需要读取 XML, 还需要写入 XML. 为此, Qt 同样提供了三种方法:

  • 使用 QXmlStreamWriter, 与 QXmlStreamReader 相对应.
  • DOM 方式, 首先在内存中生成 DOM 树, 然后将 DOM 树写入文件.
  • 使用 QString 手动生成 XML.
