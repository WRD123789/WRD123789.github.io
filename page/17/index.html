相对引用
发表于2023-11-23|笔记Git
相对引用 Git 还提供了相对引用, 即以某一个分支指针指向的提交为参照: 操作符 “^”: 把 ^ 加到引用名称的后面, 表示让 Git 寻找指定分支名指向的提交记录的父节点, 可以引用多个. 例如: main^ 相当于寻找 “main 的父节点”, main^^ 相当于寻找 “main 的父节点的父节点”. 操作符 “~”: 把 ~<num> 加到引用名称的后面, 表示让 Git 寻找指定分支名指向的提交计量的第 num 个父节点, 没有数字默认第一个父节点. 例如: main~4, 表示寻找 “main 的第四个父节点”. Git 介绍 Git 是一个分布式版本控制系统, 而不是集中式的版本控制系统, 即每个开发人员的电脑都存储整个项目的历史 (所有提交记录), 我们将整个项目的历史称为 “存储库”. Git 的主要组成部分如图所示: 对象 Git 中的对象如下所示: Blob 对象: 在 git 中存储文件内容的对象. Blob 相比普通文件不包含元数据 (例如: 文件创建时间等), Blob 只有内容 (数据的二进制流). git 中的 Blob 通过 SHA-1 哈希值唯一标识 (对于相同内容的 Blob, 即使在两台不同的电脑上, 产生的 SHA-1 哈希值仍然相同). Tree 对象: 一个目录列表, 引用其他 Blob 和 Tree 对象, 并存储其他对象对应的名字. 用 SHA-1 哈希值唯一标识, 通过其他对象的 SHA-1 引用它们. Commit 对象: git 中的一个快照, 包含指向主要 Tree 对象的指针、元数据 (提交者、提交信息、提交时间) 和该提交的所有父节点 (同样通过它们的 SHA-1 引用它们).
