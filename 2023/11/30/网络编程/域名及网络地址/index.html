域名及网络地址

DNS 服务器

由于 IP 地址相对来说比较难记, 所以一般将 IP 地址转换为易记忆、易表述的域名. 而有一种服务器就专门保存了 Web 服务器域名与 IP 地址的对应关系, 它就是 DNS 服务器.

DNS 中的域名都是用句点来分隔的, 比如 www.server.com, 这里的句点代表了不同层次之间的界限. 在域名中, 越靠右的位置表示其层级越高. 实际上, 域名最后还有一个点, 比如 www.server.com., 最后的一个点代表根域名, 所以该例中 . 根域在最顶层, 它的下一层就是 .com 顶级域, 再下面是 server.com.

所以域名的层级关系类似一个树状结构:

  • 根 DNS 服务器 (.)
  • 顶级域 DNS 服务器 (.com)
  • 权威 DNS 服务器 (如: server.com)

其中根域的 DNS 服务器信息保存在互联网中所有的 DNS 服务器中. 因此, 任何 DNS 服务器就都可以找到并访问根域 DNS 服务器.

那么, 客户端只要能够找到任意一台 DNS 服务器, 就可以通过它找到根域 DNS 服务器, 然后再一路顺藤摸瓜找到位于下层的某台目标 DNS 服务器.

域名解析的完整流程 (假设解析域名 www.test.com 如下:

  1. 客户端首先会发出一个 DNS 请求, 询问 www.test.com 的 IP 地址, 并发送给本地 DNS 服务器.
  2. 当本地 DNS 服务器收到查询请求后, 首先在浏览器 / 系统缓存中查找域名, 接着在 host 文件中查找域名, 再在本地 DNS 服务器中查找域名.
  3. 如果查不到, 可能是因为该服务器不是请求域的授权服务器, 并且以前查询的缓存中没有需要的记录, 这时本地 DNS 服务器必须让客户端向转发域名服务器 (转发域名服务器负责所有非本地域名的查询) 发送请求, 如果没有转发域名服务器则客户端会向根域名服务器 (此时本地 DNS 服务器会向客户端发送根域名服务器的 IP 地址) 发送请求.
  4. 根域名服务器收到来自本地 DNS 的请求后, 发现后置是 .com, 则会让客户端向顶级域名服务器 (此时根域名服务器会向客户端发送顶级域名服务器的 IP 地址) 发送请求.
  5. com 顶级域名服务器会向客户端提供 test.com 域服务器的 IP 地址, 此时客户端就可以通过 test.com 域服务器得到 www.test.com 的 IP 地址.

IP 地址和域名之间的转换

除非商业需要, 否则一般不会轻易改变服务器域名, 但会相对频繁地改变服务器 IP 地址. 由于开发的客户端需要使用服务端提供的服务, 所以需要将服务器的地址信息写入客户端的程序代码中, 由于 IP 地址比域名发生变更的概率更高, 所以每次运行程序时应该通过域名获取服务器的 IP 地址, 再接入服务器, 这样程序就不会依赖于服务器的 IP 地址. 所以程序中也需要用于 IP 地址和域名之间相互转换的函数.

gethostbyname 函数可以通过传递字符串格式的域名获取 IP 地址:

该函数只需要传递域名字符串, 就会返回包含地址信息的 hostent 结构体.

hostent 结构体定义如下:

  • h_name: 该变量中存有官方域名.
  • h_aliases: 由于同一个 IP 地址可以绑定多个域名, 因此, 除官方域名外还可指定其他域名. 这些信息可以通过 h_aliases 获得.
  • h_addrtype: gethostbyname 函数不仅支持 IPv4, 还支持 IPv6. 通过该变量可以获得保存在 h_addr_list 的 IP 地址的地址族信息. 若是 IPv4, 则该变量存有 AF_INET.
  • h_length: 保存IP地址长度. 若是 IPv4 地址, 因为是 4 个字节, 则保存 4; 若是 IPv6, 因为是 16 个字节, 故保存 16.
  • h_addr_list: 通过此变量以整数形式保存域名对应的 IP 地址. 对于访问量大的网站, 可能会将多个 IP 地址分配给同一个域名, 利用多个服务器进行负载均衡, 所以该变量存储的是 IP 地址列表. 若只看 hostent 结构体的定义, 结构体成员 h_addr_list 保存字符串指针数组, 但字符串指针数组中的元素实际指向的是 in_addr 结构体变量地址值而非字符串, 如下图所示:
    • h_addr_list 保存的数组类型不是 in_addr 结构体的指针数组, 而是采用了 char 指针的原因是: hostent 结构体并不只是服务于 IPv4, h_addr_list 同样可以保存 IPv6 地址信息, 因此, 考虑到通用性, 将其声明为 char 指针类型的数组.
    • 指针对象不明确时, 更适合使用 void 指针类型, 而不采用 void 指针是因为套接字相关函数都是在 void 指针标准化之前定义的.
