Java 泛型

泛型

泛型允许创建支持任何类型输入的其他.

  • 泛型能够将类型参数化, 也就是说可以把类型当作一个参数传递给一个类或者一个方法.
  • 泛型提供了一种类型检测机制, 只有类型相匹配的数据才能正常赋值, 使得软件的安全性有所提高.
  • 泛型提高了程序代码的可读性, 使得程序员能够知道代码要操作的数据类型.

  • E 表示元素 (Element) 或异常 (Exception).
  • K 表示键 (Key).
  • V 表示值 (Value), 通常与 K 一起使用.

上限与下限

设置泛型对象的上限使用 extends, 表示参数类型只能是该类型或该类型的子类:

  • 声明对象: 类名<? extends 类> 对象名
  • 定义类: 类名<泛型标签 extends 类> {}

设置泛型对象的下限使用 super, 表示参数类型只能是该类型或该类型的父类：

  • 声明对象: 类名<? super 类> 对象名

  • 定义类: 类名<泛型标签 super 类> {}

类型擦除

泛型类型只存在于代码编译阶段, 在进入 Java 虚拟机之前, 与泛型相关的信息都会被抹除, 即泛型类和普通类在 Java 虚拟机内没有区别.
