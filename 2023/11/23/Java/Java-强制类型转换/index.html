强制类型转换

Java 代码运行之前会进行类型检查, 此时编译器检查的是变量的静态类型, 此时会导致一些不易察觉的错误.

例如:

假设类 VengefulSLList 继承于类 SLList, 且 printLostItems 方法只有类 VengefulSLList 拥有.

如果从运行时的变量的动态类型来看, 变量 sl 和 vsl 的动态类型都是 VengefulSLList, 则 sl 拥有方法 printLostItems, 则 sl.printLostItems() 这一行代码应当能运行, sl 又是 VengefulSLList 类型, 所以 VengefulSLList<Integer> vsl2 = sl 这一行代码也应当可以运行,
但是上述代码在运行时却发生编译错误.

上述分析会考虑编译器在代码运行之前会先进行类型检查, 类型检查检查的是变量的静态类型, 显然编译器无法预测运行时变量的实际类型, 而变量 sl 的静态类型为 SLList, 而 SLList 没有方法 printLostItems, 所以 sl.printLostItems 这行代码会编译错误, 同样,变量 vsl2 的类型 VengefulSLList 是变量 sl 的类型的子类, 这行代码显然也会出问题.

为了解决上面的问题, Java 支持强制类型转换, 即在需要转换类型的表达式面前添加 (类型).
