不可变类型与可变类型

不可变类型

  • 不可变类型是指一旦对象被创建, 其状态就不能被更改.

    • 例如: 在 Java 中, String 类型是不可变类型, 如果我们有一个 String 变量 s, 我们对它进行赋值:

  • 在 Java 中, 基本类型如 int、double、char 等也是不可变类型. 但是基本类型的变量存储的是值本身, 而对象类型的变量存储的是引用 (指向对象的内存地址).
  • 优点
    • 不可变类型的对象是线程安全的, 因为它们不能被修改.
    • 不可变类型的对象可以被安全地共享, 它们的状态不能被修改.
  • 缺点
    • 对不可变类型的变量进行频繁修改会产生大量的临时拷贝.

可变类型

  • 可变类型是指创建的对象的状态可以修改.

    • 可变类型的变量进行频繁修改时不需要拷贝, 可以提高效率.
  • 缺点
    • 可变类型的对象不是线程安全的, 因为它们可以被修改, 所以如果同时有多个线程修改同一个对象, 可能会导致竞态条件.
    • 可变类型的对象不能被安全地共享, 因为它们的状态可以被修改.

在 Java 中, final 关键字可以用来修饰变量, 表示它们的值不可改变. 被 final 修饰的变量是常量, 即一旦赋值后就不能被修改了. 如果被修饰的变量是一个基本类型的变量, 那么该变量的值就是不可改变的; 如果被修饰的变量是一个对象类型的变量, 那么该变量存储的引用不可改变, 但对象的状态仍然可以改变.
