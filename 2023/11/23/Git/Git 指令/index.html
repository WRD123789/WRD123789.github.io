Git 指令

  1. 创建存储库

    把当前文件夹变成一个 Git 可以管理的仓库, 这个目录里面的所有文件都可以被 Git 管理, 每个文件的修改/删除, Git 都能跟踪, 以便任何时刻都可以追踪历史, 或者在将来某个时刻还可以将文件”还原”, 此操作会产生一个隐藏的 .git 文件夹, 这个文件夹是 Git 用来跟踪管理版本的, 尽量不要修改:

  2. 跟踪指定文件

    将当前文件夹内的改变存放到暂存区 (Staging Area).

    • 将文件 A1∼AnA_1 \sim A_nA1​∼An​ 加入暂存区.
    • 将所有改变的文件加入暂存区.

    git add 告诉 Git 需要跟踪哪些文件, 一个文件一旦被跟踪, 后面该文件的任何变化都会被捕捉到.

    注意: 即使一个文件已经被跟踪, 但是如果该文件发生新的变化, 仍然要使用 git add 将其加入暂存区, 因为每次提交都会清除暂存区, 只有暂存区的文件才能被提交.

  3. 提交

    将暂存区中的所有文件作为一个块提交, 其中还包含了独一无二的编号 (哈希值), 提交信息的用户, 提交表述和提交时间, 并清除暂存区的数据:

    新的提交是 HEAD 的直接子节点，同时当前分支的指针也会被更新以指向它.

      一般情况下master 指针总是指向 master 分支的最新提交, 当然也可以用命令移动 master 指针, 但是只有 HEAD 指针指向 master 指针时才能在 master 分支上添加新的提交, HEAD 指针通常情况下指向分支名

  4. 移动 HEAD 指针

    git checkout ID : 通过移动 HEAD 指针转移到指定编号 (哈希值) 的提交, 并修改文件夹中的文件使其匹配当前提交, 即 HEAD 指针控制文件夹中文件的状态 (版本).

    ID 本质上是哈希值, 且此处不一定要输入完整的哈希值, 只要提供能够唯一标识提交记录的前几个字符即可.

    git checkout 不会改变提交历史记录.

    注意: checkout 会对所有因为回滚而改变的文件自动调用 git add
