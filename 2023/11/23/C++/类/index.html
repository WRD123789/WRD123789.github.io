内联函数

内联函数是一种编译器优化技术, 用于在函数调用的地方直接将函数体插入, 而不是通过函数调用的方式进行执行, 即内联函数在编译时会将函数体复制到每个调用该函数的地方, 从而避免函数调用的开销.

内联函数通常被用于一些短小且调用频繁的函数, 它的主要目的时提高程序的执行效率, 减少函数调用的开销.

在 C++ 中, 可以通过两种方式定义内联函数:

  1. 将函数定义放在类定义的内部
    • 这样的函数通常被隐式地视为内联函数, 但编译器只会将其视为内联函数的候选, 最终是否将其内联化取决于编译器的决策
    • 编译器是否将函数内联化是基于一些因素进行评估的, 如函数的大小、复杂性、调用频率等. 如果函数体过于庞大或复杂, 编译器可能会选择不内联化该函数.
  2. 使用 inline 关键字显示声明内联函数
    • inline 关键字只是向编译器提供了一个建议, 它并不保证函数一定会被内联化.
    • 内联函数的定义一般放在头文件中, 以便在每个调用该函数的地方进行内联展开.

构造函数

C++ 中的构造函数是一种特殊的成员函数, 用于创建和初始化类的对象. 构造函数在对象创建时被调用, 用于完成对象的初始化工作, 包括成员变量的赋值、动态内存的分配、资源的初始化等操作.

构造函数具有以下特点：

  1. 构造函数的名称与类名相同, 没有返回类型, 也不需要显式声明返回类型.
  2. 构造函数可以被重载, 允许定义多个具有不同参数列表的构造函数, 以满足不同的对象初始化需求.
  3. 构造函数可以包含参数, 用于接收初始化对象所需的值.
  4. 构造函数可以具有默认参数, 这样在创建对象时可以选择性地省略部分参数.
  5. 构造函数可以是隐式调用的, 例如在对象定义时自动调用默认构造函数.
  6. 构造函数可以是显式调用的, 例如通过调用拷贝构造函数创建对象的副本.

常量成员函数

常量成员是指在类中声明和定义的函数, 其特点是在函数声明的末尾使用 const 修饰符. 常量成员函数承诺不会修改对象的状态 (即类的数据成员).

常量成员函数具有以下特性:

  1. 不修改对象的状态: 常量成员函数不能修改类的数据成员.
  2. 可以访问常量成员: 常量成员函数可以访问对象的所有成员, 包括常量成员和非常量成员.
  3. 通过常量对象调用: 常量成员函数可以通过常量对象进行调用, 以保证在调用过程中对象的状态不会被修改.

友元

友元 (friend) 是一种特殊的访问权限, 它允许一个类或函数访问另一个类的私有成员. 通过友元关系, 可以在需要访问私有成员的情况下, 实现不同类之间的协作和信息交换.

友元的主要特点有:

  1. 友元关系是通过在类的声明中使用 friend 关键字来建立的. 在类中声明的友元函数或友元类可以访问该类的私有成员.
  2. 友元关系是单向的, 不具备传递性. 如果类 A 是类 B 的友元类, 类 B 并不会是类 A 的友元类.
  3. 友元关系破坏了封装性, 因为它允许其他类或函数绕过访问控制, 直接访问类的私有成员. 因此, 需要谨慎使用友元关系, 以保持代码的安全性和可维护性.

析构

析构函数是一种特殊的成员函数, 用于在对象的生命周期结束时进行资源的清理和释放. 它与类的构造函数相对应, 在对象被销毁时自动调用.

析构函数的命名规则与构造函数相同, 使用类名前加上波浪号 (~) 作为前缀, 没有返回类型, 并且不能被重载.

析构函数在以下情况下被自动调用:

  1. 当对象的作用域结束时.
    • 例如: 当一个局部对象离开其声明的作用域时.
  2. 当一个对象被 delete 运算符释放时.
    • 例如: 在动态分配的对象销毁时.
  3. 当一个对象是容器中的元素, 而容器本身被销毁时.
    • 例如: 在一个容器对象销毁时, 它所包含的所有元素都会被销毁.

委托

类的委托 (Delegation/Composition by reference) 是指一个类将一部分或全部的成员函数调用委托给另一个类来处理. 在类的委托中, 一个类 (委托类) 可以通过维护指向另一个类 (委托给类) 的对象的指针来实现其功能, 从而避免重复编写相同的代码, 提高代码的复用性和可维护性. 相比于复合, 委托类无法管理委托给类的生命周期.

继承

继承 (Inheritance) 允许一个类 (称为子类或派生类) 从另一个类 (称为父类或基类) 继承属性和行为. 通过继承, 子类可以获取父类的所有公共成员 (成员变量和成员函数), 并且可以在此基础上添加自己特有的成员和行为.

在C++中实现继承: 为了创建继承自基类的派生类, 我们必须遵循以下语法:

  • subclass_name 是派生类的名字.
  • access_mode 是派生类的继承模式.
  • base_class_name 是基类的名字.

注意: 派生类不会继承基类对私有数据成员的访问权. 但是, 它确实继承了一个完整的基类对象, 该对象包含基类声明的任何私有成员.

在 C++ 中, 有三种继承模式:

  1. 公有继承 (public): 当派生类以 public 方式继承自基类时, 基类的公有成员也是派生类的公有成员, 基类的保护成员也是派生类的保护成员. 基类的私有成员不能直接被派生类访问, 但是可以通过调用基类的公有和保护函数来访问. 例如:

重载 new 和 delete

operator new 和 operator delete

new 和 delete 是用户进行动态内存分配和释放的操作符, operator new 和 operator delete 是系统提供的全局函数, new 在底层调用 operator new 全局函数来申请空间, delete 在底层通过 operator delete 全局函数来释放空间.

new 和 delete 实现原理

new 的原理如下:

  1. 调用 operator new 在内存上申请空间.
  2. 在申请的空间中执行构造函数, 完成对象的构造.

delete 的原理如下:

  1. 调用析构函数, 完成对象中资源的清理工作.
  2. 调用 operator delete 释放对象在内存中的空间.

new T[N] 的原理如下:

  1. 调用 operator new[] 在内存上申请空间, operator new[] 实际上调用 operator new 完成对 N 个对象空间的申请.
  2. 在申请的空间中调用 N 次构造函数.

delete[] 的原理如下所示:

  1. 执行 N 次析构函数, 完成 N 个对象中资源的清理.
  2. 调用 operator delete[] 释放空间, 实际在 operator delete[] 中调用 operator delete 来释放空间.
