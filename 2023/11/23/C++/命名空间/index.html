命名空间

定义

在 C++ 中, 命名空间 (namespace) 是一种用于组织代码的机制, 它可以防止名称冲突和命名重复. 命名空间可以将全局作用域内的一组变量、函数、类等实体组织在一起, 使得它们在不同的作用域 (不同的上下文) 中具有独立的标识符.

通过将相关的实体放置在命名空间中, 可以确保这些实体的名称不会与其他命名空间或全局作用域中的实体名称冲突, 从而提高了代码的可维护性和可读性.

std 命名空间

在老式的 C++ 中没有使用命名空间, 在引入了命名空间后, 为了使得老式代码能够继续工作, 在保留原来的库和头文件的情况下, 把原来的库复制一份, 在此基础上稍加修改, 把类、函数、宏等纳入命名空间 std 下, 就产生了新版 C++ 标准库. 这样存在了两份功能相似的库, 使用了老式 C++ 的程序可以继续使用原来的库, 新开发的程序可以使用新版的 C++ 库.

为了避免头文件重名, 新版 C++ 库也对头文件的命名做了调整, 去掉了后缀 .h, 所以老式 C++ 的 iostream.h 变成了 iostream, fstream.h 变成了 fstream, 而对于原来 C 语言的头文件, 也采用同样的方法, 但在每个名字前还要添加一个 c 字母, 所以 C 语言的 stdio.h 变成了 cstdio, stdlib.h 变成了 cstdlib.

所以对于 C++ 头文件:

  1. 旧的 C++ 头文件: 如 iostream.h、fstream.h 等将会继续被支持, 尽管它们不在官方标准中. 这些头文件的内容不在命名空间 std 中.
  2. 新的 C++ 头文件, 如 iostream、fstream 等包含的基本功能和对应的旧版头文件相似, 但头文件的内容在命名空间 std 中.
  3. 标准 C 头文件如 stdio.h、stdlib.h 等继续被支持. 头文件的内容不在 std 中.
  4. 具有 C 库功能的新 C++ 头文件具有如 cstdio、cstdlib 这样的名字. 它们提供的内容和相应的旧的 C 头文件相同, 只是内容在 std 中.

在写代码的时候尽量不使用关键字 using. 例如 using namespace std 会使得命名空间 std 中的所有实体暴露在全局作用域中, 这样可能会与当前文件中的实体的命名冲突. 由于头文件可能被其他人使用, 所以在头文件中使用关键字 using 可能给其他人添麻烦 (会在预处理阶段被替换进引用者的源文件里, 引起命名冲突), 虽然源文件中使用 using 不会影响其他人的代码, 但是为了养成良好的习惯, 尽量不使用 using.
