C++ 智能指针

智能指针

在 C++ 中, 动态内存管理经常会产生两种问题:

  1. 忘记释放堆中分配的内存, 导致内存泄漏.
  2. 在尚有指针引用堆中内存的情况下就释放了它, 这会产生引用非法内存的指针.

为了更加容易且安全地使用动态内存, 引入智能指针的概念, 智能指针的行为类似于常规指针, 只是它负责自动释放所指向的对象.

C++ 标准库提供了四种指针: auto_ptr、shared_ptr、unique_ptr、weak_ptr, 其中 C++11 只支持后三种, C++98 支持所有四个.

auto_ptr

auto_ptr 是 C++98 定义的智能指针模板, 其定义了管理指针的对象, 可以将动态分配 (通过 new 函数) 得到的地址赋给这种对象. 当对象被销毁时, 其析构函数会调用 delete 来释放动态分配的内存.

shared_ptr

std::shared_ptr 持有的资源可以在多个 std::shared_ptr 之间共享, 每多一个 std::shared_ptr 对资源的引用, 资源引用计数将增加 1, 每一个指向该资源的 std::shared_ptr 对象析构时, 资源引用计数减 1, 最后一个 std::shared_ptr 对象析构时, 发现资源计数为 0, 将释放其持有的资源. 多个线程之间, 递增和减少资源的引用计数是安全的.

std::shared_ptr 提供了一个 use_count() 方法来获取当前持有资源的引用计数. 除了上面描述的, std::shared_ptr 用法和 std::unique_ptr 基本相同.

weak_ptr

std::weak_ptr 是一个不控制资源生命周期的智能指针, 是对对象的一种弱引用, 只是提供了对其管理的资源的一个访问手段, 引入它的目的为协助 std::shared_ptr 工作.

std::weak_ptr 可以从一个 std::shared_ptr 或另一个 std::weak_ptr 对象构造, std::shared_ptr 可以直接赋值给 std::weak_ptr , 也可以通过 std::weak_ptr 的 lock() 函数来获得 std::shared_ptr. 它的构造和析构不会引起引用计数的增加或减少. std::weak_ptr 可用来解决 std::shared_ptr 相互引用时的死锁问题.

智能指针对象的大小

一个 std::unique_ptr 对象大小与裸指针大小相同 (即 sizeof(std::unique_ptr) == sizeof(void*)), 而 std::shared_ptr 的大小是 std::unique_ptr 的一倍.

在 32 位机器上, std::unique_ptr 占 4 字节, std::shared_ptr 和 std::weak_ptr 占 8 字节; 在 64 位机器上, std::unique_ptr 占 8 字节, std::shared_ptr 和 std::weak_ptr 占 16 字节.
