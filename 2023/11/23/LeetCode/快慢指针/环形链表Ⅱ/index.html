LeetCode142. 环形链表 Ⅱ

题目链接

https://leetcode.cn/problems/linked-list-cycle-ii/

题目描述

给定一个链表的头节点  headheadhead , 返回链表开始入环的第一个节点.  如果链表无环, 则返回 nullnullnull.

如果链表中有某个节点, 可以通过连续跟踪 nextnextnext 指针再次到达, 则链表中存在环. 为了表示给定链表中的环, 评测系统内部使用整数 pospospos 来表示链表尾连接到链表中的位置 (索引从 000 开始) . 如果 pospospos 是 −1-1−1, 则在该链表中没有环. 注意: pospospos 不作为参数进行传递, 仅仅是为了标识链表的实际情况.

不允许修改链表.

算法过程

  • 快慢指针

  1. 双指针第一次相遇

    设开始时慢指针 slowslowslow 和快指针 fastfastfast 指向链表的头节点 HEADHEADHEAD, 慢指针每次移动 111 个单位, 快指针每次移动 222 个单位.

    1. 链表无环, 则 fastfastfast 指针走到链表末端, 此时返回 nullnullnull.
    2. 链表有环, 因为快指针比慢指针每次多走一个单位, 即快指针和慢指针之间的距离每次增加一个单位, 所以如果链表有环, 快指针一定可以追上慢指针. 假设快指针和慢指针第一次相遇时慢指针移动了 sss 个单位, 则快指针移动了 fff 个单位, 假设链表共有 a + ba \ + \ ba + b 个节点, 其中链表头部到环的入口处有 aaa 个节点. 显然当快指针和慢指针相遇时, 快指针比慢指针多走了 nnn 个环的长度, 即 f = n⋅b+sf \ = \ n \cdot b + sf = n⋅b+s, 又因为 f = 2⋅sf \ = \ 2 \cdot sf = 2⋅s, 所以 s = n⋅bs \ = \ n \cdot bs = n⋅b, f = 2⋅n⋅bf \ = \ 2 \cdot n \cdot bf = 2⋅n⋅b.

  2. 问题分析:

    • 如果指针从链表的 HEADHEADHEAD 指针处开始, 止步于环的入口处, 则总的步数 k = a + m⋅bk \ = \ a \ + \ m \cdot bk = a + m⋅b.
    • 在快慢指针第一次相遇后, 慢指针已经走了 n⋅bn \cdot bn⋅b 步, 所以只要让慢指针再走 aaa 步就能够找到环的入口的位置.
    • 如何确保慢指针刚好走 aaa 步? 可以在 HEADHEADHEAD 处再指定一个指针, 该指针和慢指针一样每次移动 111 个单位, 在慢指针和快指针第一次相遇后出发, 当慢指针和该指针相遇时, 慢指针正好走完 aaa 步.

  3. 双指针第二次相遇

    双指针第一次相遇后, 保持慢指针的位置不变, 将快指针移动到链表的头节点 HEADHEADHEAD 指针处 (节省空间, 此时 fastfastfast 指针已无用处, 不必多指定一个指针), 此时 slowslowslow 和 fastfastfast 每次只移动一个单位, 当两指针相遇时, 便是环的入口所在的位置, 直接返回 slowslowslow 所指向的节点即可.

时间复杂度分析

慢指针总共移动了 a + n⋅ba \ + \ n \cdot ba + n⋅b 步, 快指针总共移动了 a + 2⋅n⋅ba \ + \ 2 \cdot n \cdot ba + 2⋅n⋅b 步, 总的移动步数为 2⋅a + 3⋅n⋅b2 \cdot a \ + \ 3 \cdot n \cdot b2⋅a + 3⋅n⋅b, 其中 nnn 与输入量无关, 所以总的时间复杂度为 O(N)O(N)O(N).
