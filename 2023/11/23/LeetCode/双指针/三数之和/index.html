LeetCode 15. 三数之和

题目链接

https://leetcode.cn/problems/3sum/

题目描述

给你一个包含 n 个整数的数组 nums, 判断 nums 中是否存在三个元素 a, b, c.使得 a + b + c = 0 ? 请你找出所有和为 0 且不重复的三元组.

注意：答案中不可以包含重复的三元组.

示例 2:

示例 3:

  • 0 ≤\leq≤ nums.length ≤\leq≤ 3000
  • -10510^{5}105 ≤\leq≤ nums[i]nums[i]nums[i] ≤\leq≤ 10510^{5}105

题解

算法流程

  • 排序, 双指针
  • 对数组按升序排序
  • 遍历排序后的数组 nums:
    • 若 nums[i]nums[i]nums[i] > 0: 因为数组是有序的, 所以后面不可能再有三个数相加和等于 0, 可以直接返回结果.
    • 对于重复元素直接跳过, 避免出现重复解.
    • 设置左指针 left_p = i + 1, 右指针 right_p = n -1, while 循环, 直到两指针相遇:
      • 如果 nums[i]nums[i]nums[i] + nums[left_p]nums[left\_p]nums[left_p] + nums[right_p]nums[right\_p]nums[right_p] = 0, 则对左右指针分别判断是否有重复项, 有则剔除.
      • 如果 nums[i]nums[i]nums[i] + nums[left_p]nums[left\_p]nums[left_p] + nums[right_p]nums[right\_p]nums[right_p] < 0, 说明和比较小, 移动左指针来增加和.
      • 如果 nums[i]nums[i]nums[i] + nums[left_p]nums[left\_p]nums[left_p] + nums[right_p]nums[right\_p]nums[right_p] > 0, 说明和比较大, 移动右指针来减小和.

时间复杂度分析

  • 对数列排序所需的时间复杂度为 O(Nlog⁡2NN\log_2^{N}Nlog2N​), 遍历数组所需时间复杂度为 O(N), 双指针遍历所需时间复杂度为 O(N), 则总的时间复杂度为 O(Nlog⁡2NN\log_2^{N}Nlog2N​) + O(N2N^{2}N2) = O(N2N^{2}N2).
