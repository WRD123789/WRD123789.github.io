常用术语

发表于2023-11-24|更新于2023-11-24|笔记计算机组成原理
|字数总计:1.6k|阅读时长:5分钟

术语

  • CPU: Central ProcessingUnit, 中央处理器, 是计算机硬件的核心部件, 主要由运算器和控制器组成.
  • PC: Program Counter, 程序计数器, 其功能是存放当前欲执行指令的地址, 并可自动计数形成下一条指令地址.
  • IR: Instruction Register, 指令寄存器, 其功能是存放当前正在执行的指令.
  • CU: Control Unit, 控制单元(部件), 为控制器的核心部件, 其功能是产生微操作命令序列.
  • ALU: Arithmetic Logic Unit, 算术逻辑运算单元, 为运算器的核心部件, 其功能是进行算术、逻辑运算.
  • ACC: Accumulator, 累加器, 是运算器中既能存放运算前的操作数, 又能存放运算结果的寄存器.
  • MQ: Multiplier-QuotientRegister, 乘商寄存器, 乘法运算时存放乘数、除法时存放商的寄存器.
  • X: 此字母没有专指的缩写含义, 可以用作任一部件名, 在此表示操作数寄存器, 即运算器中工作寄存器之一, 用来存放操作数；
  • MAR: Memory AddressRegister, 存储器地址寄存器, 在主存中用来存放欲访问的存储单元的地址.
  • MDR: Memory Data Register, 存储器数据缓冲寄存器, 在主存中用来存放从某单元读出、或要写入某存储单元的数据.
  • I/O: Input/Outputequipment, 输入/输出设备, 为输入设备和输出设备的总称, 用于计算机内部和外界信息的转换与传送.
  • MIPS: Million Instruction PerSecond, 每秒执行百万条指令数, 为计算机运算速度指标的一种计量单位.
  • 命中: 在缓存中找到地址.
  • 未命中: 未在缓存中找到地址.
  • 刷新: 将所有缓存行标记为无效, 将缓存中的所有脏缓存行写入内存中.
  • 命中率: 在缓存中实际命中的访问的比例.
  • 未命中率: 1 - 命中率.
  • 未命中惩罚: 将块从存储层次的较低级别中加载到缓存中所需要的时间.
  • 命中时间: 访问缓存所花费的时间.
  • 未命中时间: 未命中惩罚 + 命中时间.
  • 缩写: 用 ‘$’ 代表缓存.
  • 平均内存访问时间 (AMAT):

    • 命中时间 + 未命中率 × 未命中惩罚.

    • 对于一个 n-级 缓存系统:

      AMAT = Time for a L1L_1L1​$ hit + (Local) Miss rate L1L_1L1​$ × (Time for a L2L_2L2​$ hit + (Local) Miss rate × … (Time for a LnL_nLn​$ hit + (Local) Miss rate LnL_nLn​$ × LnL_nLn​$ Miss penalty))

  • 缓存级别: 存储层次结构中的顺序, 其中L1$ 是最接近处理器的, L1 缓存只能包含数据 (数据缓存, D$) 或指令 (指令缓存, I$), 而大多数 L2+ 缓存都是统一的, 可以包含指令或数据.
  • 缓存容量: 缓存的总字节数 (不包含标记位和有效位), 缓存容量 = 关联性 × 集合数 × 缓存行大小.
  • 缓存行或缓存块: 缓存中的单个条目, 是缓存中的基本存储单元.
  • 缓存行大小或块大小: 每一个缓存行的字节数, 指其包含的数据块的大小, 不包含有效位和标记位.
  • 缓存行数: 缓存容量 / 缓存行大小.
  • 缓存关联度:
    • 给定地址可能存在的缓存行数.
    • 检查缓存是否命中所需的比较次数.
    • 直接映射: 数据元素只有一种可能的位置.
    • N 路集合关联映射: 数据元素可以位于 N 个位置中的一个.
    • 全关联映射: 数据元素可以在缓存中的任何位置.
  • 缓存集合数: 缓存行数 / 缓存关联度
  • 全局未命中率:
    • 多级缓存中的某一级缓存相对于总的内存访问次数的未命中率.
    • 全局未命中率 = 该级缓存的未命中次数 / CPU 访问内存的总次数.
    • 全局未命中率Ln_{L_n}Ln​​ = 局部未命中率L1_{L_1}L1​​ × … × 局部未命中率Ln_{L_n}Ln​​.
  • 局部未命中率:
    • 多级缓存中某一级缓存相对于到达该级缓存的访问次数的未命中率.
    • 局部未命中率 = 该级缓存的未命中次数 / 到达该级缓存的访问次数.
    • 局部未命中率Ln_{L_n}Ln​​ = 全局未命中率Ln_{L_n}Ln​​ / 全局未命中率Ln−1_{L_{n-1}}Ln−1​​
  • 核心: 也称内核, 是一个独立的执行单元, 根据处理器的不同, 可以一次运行 N 个线程 (例如普通的 Intel 和 AMD 芯片每个核有两个线程, Raspberry Pi 4 每个核只有一个线程).
  • 进程: 一段程序的执行过程, 进程是一个独立的存储空间, 用虚拟存储空间分隔它, 线程在进程中进行, 一个进程至少拥有一个执行线程. 在串行处理器中, 一个进程运行一个执行线程; 在多处理器中, 在同一个进程中运行多个线程.
  • 线程: 执行一些任务的顺序指令流, 多个线程占用相同的进程和相同的存储空间, 将软件线程映射到可用的硬件线程取决于操作系统.
  • 操作系统线程:
    • 基于硬件线程复用多个软件线程.
    • 通过在时间上复用多个硬件线程, 给与你一种有许多线程在同时活跃的假象.
    • 在中断线程时, 将它的寄存器和程序计数器保存到内存中.
    • 在恢复线程时, 加载对应的寄存器并跳转到对应的程序计数器上.
  • 硬件多线程:
    • 在缓存未命中时, 硬件切换线程以在等待缓存未命中的时候做其他工作.
    • 放入冗余硬件 (多个程序计数器、多个寄存器堆等), 这样就无需保存每个线程的上下文.
  • RAM:
    • 随机存取存储器, 也叫主存, 能包含代码和数据.
    • 为了访问数据或代码, 需要将它们从硬盘中加载到 RAM 中.
  • 页面:

    • 为了使磁盘和 RAM 之间的数据传输更容易, 我们对移动数据的粒度 (也称为页面) 进行标准化.

    • 页面的大小通常为 4 KB.

    • 这意味着如果我们想要访问步骤 RAM 中的数据, 需要将数据所在的整个页面加载到 RAM 中.
