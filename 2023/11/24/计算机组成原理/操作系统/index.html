操作系统

操作系统的任务

  • 为硬件提供简便的抽象

    • 我们通过操作系统来访问硬件, 而不是直接与硬件交互

      • 处理器被抽象为线程, 进程通过线程来和处理器进行交互.
      • 缓存和内存被抽象为地址空间 (Address Spaces).
      • 硬盘被抽象为文件.
      • 网络设备被抽象为 Sockets, 可以通过它来和网络进行交互.

    • 我们可以在一个机器上运行多个进程, 每个编译的程序都有自己的进程, 每个进程都有自己的线程、地址空间、文件和 Sockets.

  • 管理、隔离、保护和分享资源

    • 由操作系统进行上下文切换.

    • 每个进程都有操作系统分配的地址空间, 如果某个进程非法访问了不属于它的地址空间, 就会发生 Segment Fault:

      此时会停止该进程, 并让其他进程继续在处理器上运行.

    • 操作系统会将自己以及其他进程相互隔离来保护它们和自己.

  • 提供与外部世界交互的手段.

  • 分隔不同的进程.

操作系统双模式操作

  • 内核模式: 能够直接和硬件交互, 写入内核内存.
  • 用户模式: 某些操作是禁止的, 比如直接和硬件交互、写入内核内存.

操作系统需要做的大多数工作不需要处于内核模式, 所以操作系统大多数情况下在用户模式下运行, 用户模式下工作能够防止产生严重的错误 (例如: 导致整个操作系统崩溃).

在用户模式下, 如果我们的程序因为某些原因需要在内核模式下运行, 就需要将控制权交给操作系统, 如果操作系统执行用户想要执行的操作, 然后再将控制权还给用户, 因此用户永远不能亲自在内核模式下操作.

执行程序

  1. 加载器将程序加载到内存中.
  2. 加载器设置 argc 和 argv.
  3. 操作系统跳转到 main 函数并将控制权交给程序.

上下文切换

从一个正在执行的进程 (程序) 切换到另一个进程的过程.

允许多个进程在同一个处理器上运行.

由操作系统决定何时切换上下文.

过程:

  1. 操作系统从当前的进程中接收 CPU 的控制权.
  2. 操作系统将当前进程的状态保存.
  3. 操作系统加载下一个进程的状态.
  4. 操作系统将 CPU 的控制权交给下一个进程.

系统调用

允许程序向操作系统请求服务.

例如:

  • 新建 / 删除文件
  • 读 / 写文件
  • 访问外部设备, 如扫描仪、打印机
  • RISC 中的 ecalls

与函数调用类似, 只不过是在内核模式下执行.

系统调用时会发生模式切换:

例如:

假设正在用户模式下执行一条指令 is−1i_{s - 1}is−1​

当执行到下一条 isi_{s}is​ 时发生系统调用

此时会切换到内核模式, 让操作系统来处理这个调用

操作系统完成调用后, 切换回用户模式

中断与异常

中断

  • 由当前运行程序的外部的事物引起.
  • 例如: 在播放视频时按下空格键来暂停.
  • 对于当前程序来说是异步的, 所以不需要立即处理, 但需要在合适的时间内处理.

异常

  • 由当前程序执行期间的事件引起.
  • 例如: 非法指令、除 0
  • 对于当前程序来说是同步的, 必须立即处理, 否则可能会破坏程序.

在五级流水线处理器中, 有很多地方都有可能发生异常:

处理

  • Trap Handler: 专门用来处理中断和异常的程序.

  • 过程

    1. 对于所有错误指令之前的指令, 通过暂停来保证它们完成.
    2. 在错误指令之后的指令必须被刷新 (移除).
    3. 错误的指令同样需要被刷新.
    4. 开始执行 Trap Handler.
      1. 保存当前程序的状态 (保存所有寄存器和程序计数器).
      2. 确定导致中断和异常的原因.
      3. 处理中断和异常.
    5. 此时有两种选择

      • 恢复程序状态, 归还控制权给程序.

      • 终止程序 (释放资源等), 调度新的程序.

  • 中断和异常处理可能会在一条指令和它的下一条指令之间产生较大的间隙 (花费时间来处理中断和异常).

  • 中断和异常处理可能会使缓存在短时间内变慢 (因为 Trap Handler 本身也是程序, 也会使用缓存, 可能会驱逐出部分缓存行).

内核

内核是操作系统的核心部分.

内核负责管理资源 (如调度进程、内存资源、输入输出、外部设备).

如网络、用户界面之类的东西不在内核中.

不同的操作系统中内核所包含的东西存在差异, 上述只是一般分类.

Shell

一个公开操作系统服务的程序.

Shell 能够让你访问操作系统服务.

Boot

  1. 运行 BIOS (基本输入 / 输出系统).
    • BIOS 会运行开机自检程序 (Power-on-self-test, POST), 测试机器, 确保它能正常工作.
    • BIOS 会找到并执行 Bootloader 程序.
  2. Bootloader 加载部分操作系统.
  3. 操作系统继续加载操作系统的更多部分, 初始化服务、驱动等.
  4. 启动一个在循环中等待输入的进程 (操作系统在电脑关机前一直运行).

这个过程实际上是 Bootstrapping 的一种形式.

Bootstrapping: 一系列调用链, 在每个阶段, 向加载一个小而简单的程序, 然后在下一阶段加载大而复杂的程序.
