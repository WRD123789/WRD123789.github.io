单链表

节点

如下图所示:

裸数据结构, 每个节点 item 存储值, next 指向下一个节点, 尾节点的下一个节点为 null.

主体

如下图所示:

其中 SLList 类充当用户与 IntNode 之间的中介, 这样可以隐藏内部细节, 并且只保留简单的接口, 使用户能够更容易地使用这个单链表.

  • 将 sentinel 和 size 设置为 private 是为了限制用户的访问权限, 防止用户破坏链表.

  • 使用 size 属性来缓存链表的大小, 虽然会增大少量内存消耗, 但是计算链表的大小的速度大幅度增长.

  • 使用前哨节点 sentinel, 对于创建空列表的情况, 无需调整其他方法, 即保持所有情况的一致性, 减少代码阅读的复杂度.

    例如: 如果直接让 sentinel = null, 则之后使用 addLast 之类的方法时会报错, 不得不增加特判条件来避免错误, 而这会增加程序阅读的复杂度.

双链表

节点

如下图所示:

使用指向下一个节点的 next 指针和指向上一个节点的 prev 指针, 使双链表的 addLast 和 getLast 方法的时间复杂度从 O(n) 优化到 O(1), 但也适当增加了代码的复杂度.

主体

如下图所示:


和单链表一样, DLList 作为用户和 IntNode 的中介, 隐藏实现细节并防止用户的错误操作, 采用哨兵节点来降低代码复杂度, 但创建空列表时, 哨兵节点的 next 指针和 prev 指针都指向自己.

为什么采用双向指针而不是单独增加一个尾指针: 只是增加一个尾指针, 虽然能够优化 getLast 和 addLast 方法的时间复杂度, 但是 removeLast 方法的时间复杂度仍为 O(n).
