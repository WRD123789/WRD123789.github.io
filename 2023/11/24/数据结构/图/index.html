深度优先遍历

调用 dfs 至多 V 次 (每个标记过的节点不会调用 dfs), 所以调用 dfs 的时间复杂度为 O(V)O(V)O(V) (为什么不是 θ(V)\theta(V)θ(V)? 因为一个节点只有和搜索起点 s 之间有路径时才会被搜索, 所以并不是每个节点都会被搜索).

由于有 E 条边, 每条边最多搜索 2 次, 所以 for 循环最多执行 2⋅E2 \cdot E2⋅E 次, 由于并不是每条边都会被搜索到 (和搜索起始点 s 无连接的节点之间的边无法被搜索到), 所以 for 循环遍历边的时间复杂度为 O(E)O(E)O(E).

所以总的时间复杂度为 O(V+E)O(V + E)O(V+E).

宽度优先遍历

和深度优先遍历类似, 每个节点至多进队列 1 次, 每条边至多遍历 2 次, 所以总的时间复杂度为 O(V+E)O(V + E)O(V+E).

Dijkstra 算法

Dijkstra 算法一般用来解决最短路问题.

Dijkstra 算法允许图中有回路, 但所有边的权值必须是非负数.

最短路问题的结果必然是一棵树.

松弛: 选择一条更好的边, 即选择当前最优解.

使用一个优先队列 PQ 来查找未标记节点中距离源点最近的节点, 优先队列更快.

过程:

  • PQ.add(source, 0)
  • For other vertices v, PQ.add(v, +∞+\infty+∞)
  • while PQ is not empty:
    • p = PQ.removeSmallest()
    • Relax all edges from p

松弛边 p -> q, 其中边的权重为 w:

  • If distTo[p] + w < distTo[q]:
    • distTo[q] = distTo[p] + w
    • edgeTo[q] = p
    • PQ.changePriority(q, distTo[q])

不变量:

  • edgeTo[v] 是节点 v 的已知最优前置节点.
  • distTo[v] 是从源点到 v 的已知最优距离.
  • PQ 按 distTo 的顺序包含所有未访问的节点.

性质:

  • 始终按照与源点的总距离的顺序从小到大访问节点.
  • 松弛已经访问过的节点总是失败, 即已经访问过的节点的 distTo 和 edgeTo 已经是最优解.

拓扑排序主要解决节点依赖性问题.

拓扑排序主要有两种实现方式: 后序 DFS 和 Kahn 算法.

后序 DFS

  1. 从所有入度 (指向当前节点的边的数量) 为 0 出发进行后序 DFS, 不清除之前遍历已经标记过的节点.
  2. 翻转之前遍历得到的点集, 即为拓扑序.

上述问题使用后序 DFS 过程如下:

即后序 DFS 遍历所得到的点集为 [7, 4, 1, 3, 0, 6, 5, 2].

则拓扑排序后的点集为 [2, 5, 6, 0, 3, 1, 4, 7].

其实该算法不一定需要从入度为 0 的节点出发, 可以从任意点出发进行后序 DFS, 如果有节点未被标记, 则继续以该节点为起点进行后序 DFS, 直到所有节点被标记.

Kahn 算法

将所有入度为 0 的点排在前面, 并在原图中将它们删除, 删除后部分节点的入度减少了, 此时又会产生新的入度为 0 的节点 (如果没有产生新的入度为 0 的节点, 说明该图有环), 直到所有节点都被删除.

上述问题使用 Kahn 算法过程如下:

可以用队列来存储入度为 0 的点.

预计算所有节点的入度所需的时间复杂度为 O(E), 其中 E 为边的数量, 将入度为 0 的点入队和出队所需的时间复杂度为 O(V), 所以总的时间复杂度为 O(V + E).

DAG 最短路径树算法

Dijkstra 算法无法处理有负权边的图, 对于负权有向无环图, 可以用 DAG 最短路径树算法.

可以考虑以拓扑序访问图中的节点, 对于每次访问, 松弛其向外的边, 因为按照拓扑序来访问, 所有能够到达该节点的节点的最短路径已经计算完成, 不会发生更改, 此时负权边就不会有影响.

拓扑排序的时间复杂度为 O(V + E), 第二步每个节点最多访问一次, 每条边最多也只访问一次, 所以时间复杂度也为 O(V + E), 所以总的时间复杂度为 O(V + E).
