数组

基本数组

数组是一种由一系列编号的内存框组成的对象.

数组的第 i 个元素可以通过使用 A[i] 得到.

数组有一个固定的长度 N, N 个内存框的数据类型相同, 编号为 0 ∽\backsim∽ N.

一维数组

  • 创建一个指定长度的数组, 并用默认值填充每个内存框.
  • 根据所列元素的数量创建一个特定长度的数组, 并将所列元素从左到右依次放入每个内存框中.
  • 与第二种创建数组的方法类似, 省略 new, 但声明的数据类型要与所列元素的数据类型一致.

二维数组

  • 二维数组实际上只是数组的数组.
  • 创建一个指定长度 (例中为 3 ×\times× 3) 的数组, 并用默认值填充每个内存框.
  • 根据所列元素的数量创建一个特定长度的数组, 并将所列元素从左到右依次放入每个内存框中.
  • 与第二种创建数组的方法类似, 省略 new, 但声明的数据类型要与所列元素的数据类型一致.

可变数组

  • 数组相比于链表的优势:
    • 能够在 O(1) 的时间复杂度下得到特定位置的元素, 而链表需要 O(N) 的时间复杂度.
    • 链表不仅要存储元素, 还要存储指向下一个元素或上一个元素的指针, 所以链表所需要的空间要比数组大很多.
  • 数组相比链表的劣势:
    • 数组如果需要在中间插入元素, 则需要移动插入位置到结尾的所有元素, 而链表则不需要.
    • 数组的大小是固定的, 就算实现可变, 调整大小也需要额外的时间, 而链表可以无限大.
  • 数组实现可变的原理: 当数组中所有的内存框都被填满时, 即 size == items.length 时, 通过 resize 方法调整数组的大小, 上述代码中是将长度增加 1 倍.

    • resize 通过重新创建一个更大的数组, 使用 System.arraycopy 方法将原数组中的元素拷贝到新创建的数组中, 并使类的 items 成员指向新数组, 来舍弃原数组.

    • 为什么每次添加元素的时候长度不是增加 1, 而是乘 2: 如果每次添加元素的时候长度增加 1, 则在连续使用 addLast 方法时会不断调用 resize 方法, 而 resize 方法中的创建数组和拷贝数组也需要一定时间, 且随着数组中元素的增多而增多, 所以 resize 的时间开销随着数组元素的增大而大幅度增大, 只能通过减少 resize 的使用次数, 即每次数组满的时候多开一点空间来提高程序的效率.
