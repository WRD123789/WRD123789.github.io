大多数情况下, 我们只关心 N 非常大的时候的情况, 因为当 N 很小的时候, 计算机处理数据的速度很快, 算法的好坏不会有直观的感受, 而当 N 很大的时候, 不同算法处理相同规模的数据所花费的时间区别很大.

该方法仍然存在缺点, 计算每个简单操作的执行次数相当花时间, 且比较复杂.

增长率

随着输入数据量 N 的增加, 程序的运行时间增长的速度叫作该程序的运行时间的增长率.

一个好的算法并不一定在任何时候都比差的算法快, 例如: 一个算法运行时间为 500⋅N500 \cdot N500⋅N, 另一个算法的运行时间为 N2N^2N2, 两者的运行时间随 N 的增加而改变的图像如下:

开始时第一个算法的运行时间更长, 但随着 N 的增加, 第二个算法的运行时间快速增长, 当 N 大于某一特定值时, 第二个算法的运行时间大于第一个算法的运行时间, 所以用算法的运行时间的增长率来判断算法的优劣更为合适.

成本模型

成本模型通过简化计算操作次数的结果得到:

  • 只需要考虑程序最坏的情况, 因为考虑程序的最好情况或者平均情况是毫无意义的.
  • 当 N 很大时, 对最终时间影响最大的项应当是包含 N 的最大项, 例如: N3N^3N3 比 N2N^2N2 对结果的影响更大, N⋅logNN \cdot log^NN⋅logN 比 N 对结果的影响更大, 此时其余项对程序的最终运行时间的影响可以忽略不记.
  • 当 N 很大时, 最大项的系数相对于 N 部分对最终时间影响有限, 如果还要考虑最大项的系数, 会增加不少麻烦, 所以同样可以省略.

那么便可以把去掉系数的最大项作为该程序的运行时间的增长率, 也叫作该程序的成本模型.
