强制类型转换

隐式类型转换是编译器利用已知的转换规则 (如内置的转换规则、用户自定义的转换构造函数和类型转换函数等), 自动将一种数据类型转换为另一种数据类型, 而无需程序员明确指定转换操作.

而强制类型转换分为 C 风格的和 C++ 风格的. C 风格的强制类型转换如下所示:

虽然这种强制类型转换比较简单, 但是存在不少缺点:

  • 类型安全性: 它可以在任意类型之间转换. C 风格的强制类型转换在编译时不提供足够的类型安全性检查. 因为 (type) 形式的转换是一种强制性的位级别操作, 它无法检查转换是否合理或是否可能导致运行时错误.
    • 如: 你可以将一个指向 const 对象的指针转换为指向非 const 对象的指针, 此时编译器不会报错, 但是在运行时可能导致意想不到的错误.
  • 可读性: C 风格的强制类型转换在代码中可能不够清晰, 因为 (type) 形式的东西在 C++ 程序中有很多.

因此一般情况下尽量使用 C++ 风格的强制类型转换运算符, 只有在与旧有代码或 C 语言接口交互的情况下, 才使用 C 风格的强制类型转换.

C++ 风格的强制类型转换运算符主要有 4 种, 如下所示.

static_cast

static_cast 主要用于静态类型转换, 不能用于两个不相关类型的转换.

reinterpret_cast

reinterpret_cast 操作符通常为操作数的位模式提供较低层次的重新解释, 用于将一种类型转换为另一种不同的类型. 该操作符执行的位级别转换是不安全的, 应当谨慎使用.

const_cast

const_cast 用于删除变量的 const 属性. 但是你不能在对 const 修饰的变量调用 const_cast 之后再修改它的值, 这会导致未定义的行为.

dynamic_cast

dynamic_cast 可以将多态基类 (包含虚函数的基类) 的指针或引用强制转换为派生类的指针或引用, 而且能够检查转换的安全性. 对于无效的指针转换, 转换结果返回 nullptr 指针 (对于指针) 或抛出 std::bad_cast 异常 (对于引用).

注意: 只有基类包含虚函数时, 使用 dynamic_cast 才能正常工作. 虚函数使得类成为多态类型, 同时也确保了每个类都有一个虚函数表, 其中存储了运行时类型信息. 在没有虚函数的情况下, 编译器可能无法生成运行时类型信息, 因为缺少虚函数表, 而 dynamic_cast 需要借助 RTTI 来检测, 这就依赖于运行时类型信息的可用性.

向上转型

向上转型是将子类对象的指针或引用转换为父类对象的指针或引用. 向上转型只要确保待转换的两个类型之间存在继承关系即可. 因为向上转型始终是安全的, 所以 dynamic_cast 不会进行任何运行期间类型检查, 这个时候的 dynamic_cast 和 static_cast 就没有什么区别了.

向下转型

向下转型是将父类对象的指针或引用转换为子类对象的指针或引用, 由于这存在风险, 因此 dynamic_cast 会借助 RTTI (运行时类型识别) 进行类型检查. 此时基类必须包含虚函数, 原因见上文.

如果父类的指针 (或引用) 指向的是一个父类对象, 那么将其转换为子类的指针 (或引用) 是不安全的, 因为转换后可能会访问到子类的资源, 而这个资源是父类对象所没有的; 如果父类的指针 (或引用) 指向的是一个子类对象, 那么将其转换为子类的指针 (或引用) 则是安全的.
